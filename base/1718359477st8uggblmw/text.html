# Разница между pip freeze и pip list в Python

Разница между pip freeze и pip list заключается в том, что:

  • pip freeze выводит установленные пользователем пакеты в формате требований, который может быть использован для создания requirements.txt файла.
  • pip list выводит все установленные пакеты, включая редактируемые.

# pip freeze используется для создания requirements.txt файла

Чаще всего вы будете использовать pip freeze команду для создания requirements.txt файла.

оболочка

pip freeze > requirements.txt

pip3 freeze > requirements.txt



Команда перенаправляет выходные данные pip freeze в файл с именем requirements.txt.

Файл requirements.txt можно использовать для установки пакетов и воссоздания среды.

оболочка

pip install -r requirements.txt

pip3 install -r requirements.txt



requirements.txt Файл может быть сгенерирован только из выходных данных команды pip freeze, а не из выходных данных pip list.

Названия пакетов и конкретные версии разделены двумя знаками равенства, которые являются синтаксисом, используемым pip для установки определенной версии пакета.

оболочка

pip install requests==2.28.0

pip3 install requests==2.28.0



# Команда pip list выводит пакеты в другом формате

Команда pip list также показывает установленные пакеты и их версии, но в другом формате.

pip freeze Команда не выводит пакеты, от которых по умолчанию зависит pip, например, wheel и setuptools, в то время как pip list команда выводит.

Вы можете использовать --all опцию, если вам нужно включить пакеты pip, setuptools, distrubute и wheel в выходные данные pip freeze.

оболочка

pip freeze --all



pip list выводит все установленные пакеты, включая редактируемые, тогда как pip freeze показывает пакеты, установленные пользователем, и их зависимости.

Если вы не используете виртуальную среду, рекомендуется создать ее, поскольку они значительно упрощают управление пакетами.

оболочка

# 👇️ optionally store currently installed packages in a file

pip freeze > requirements.txt

pip3 freeze > requirements.txt


# 👇️ Use the correct version of Python when creating VENV

python -m venv venv


# 👇️ Activate on Unix or MacOS

source venv/bin/activate


# 👇️ Activate on Windows (cmd.exe)

venv\Scripts\activate.bat


# 👇️ Activate on Windows (PowerShell)

venv\Scripts\Activate.ps1


# 👇️ Upgrade pip

pip install --upgrade pip


# 👇️ Install the package in your virtual environment

pip install requests


# 👇️ Optionally install packages from a `requirements.txt` file

pip install -r requirements.txt

pip3 install -r requirements.txt



Если python -m venv venv команда не работает, попробуйте одну из следующих команды:

  • python3 -m venv venv
  • py -m venv venv

Убедитесь, что используете правильную команду активации в зависимости от вашей операционной системы.

Ваша виртуальная среда будет использовать версию Python, которая использовалась для ее создания.

Обратите внимание, что название requirements.txt - это просто соглашение. Вы можете использовать любое для хранения выходных данных pip freeze команды, и у вас может быть столько requirements файлов, сколько необходимо.
