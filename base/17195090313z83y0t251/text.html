Как исправить ModuleNotFoundError: в Python нет модуля с именем «xlsxwriter»

Натан Себастьян

Опубликовано 24 апреля 2023 г.

Время чтения: 5 минут.

При попытке работать с файлами Excel в Python вы можете получить следующую ошибку:

ModuleNotFoundError: No module named 'xlsxwriter'



Эта ошибка возникает, когда Python не может найти xlsxwriterбиблиотеку в текущей среде.

В этом руководстве показан пример, вызывающий эту ошибку, и способы ее устранения.

Как воспроизвести ошибку

Предположим, вы хотите использовать xlsxwriterмодуль для программного доступа к листам Excel с помощью Python.

Вы импортировали xlsxwriterмодуль и попытались создать книгу и лист:

import xlsxwriter

workbook = xlsxwriter.Workbook('example.xlsx')

worksheet = workbook.add_worksheet()



Но при запуске кода вы получаете следующую ошибку:

Traceback (most recent call last):

File "main.py", line 1, in <module>

import xlsxwriter

ModuleNotFoundError: No module named 'xlsxwriter'



По моему опыту, это ModuleNotFoundErrorпроисходит, когда Python не может найти модуль, который вы пытаетесь импортировать. Модуль xlsxwriterне предоставляется по умолчанию, поэтому вам нужно установить его с помощью pip.

Как исправить эту ошибку

Чтобы устранить эту ошибку, вам необходимо установить модуль, xlsxwriterкак pipпоказано ниже:

pip install xlsxwriter

# For pip3:

pip3 install xlsxwriter



После установки модуля вы сможете запускать импортирующий код xlsxwriterбез получения ошибки.

Установите команды для других сред

Команда установки может отличаться в зависимости от того, какую среду вы использовали для запуска кода Python.

Вот список распространенных команд установки в популярных средах Python для установки модуля xlsxwriter:

# if you don't have pip in your PATH:

python -m pip install xlsxwriter

python3 -m pip install xlsxwriter

# Windows

py -m pip install xlsxwriter

# Anaconda

conda install xlsxwriter

# Jupyter Notebook

!pip install xlsxwriter



После установки модуля вы сможете запустить код, не получая этой ошибки.

Другие распространенные причины этой ошибки

Если вы по-прежнему видите ошибку даже после установки модуля, это означает, что модуль xlsxwriterне найден в вашей среде Python.

Есть несколько причин, по которым может возникнуть эта ошибка:

  1. В вашей системе может быть установлено несколько версий Python, и вы используете версию Python, отличную от той, в которой установлен xlsxwriter.
  2. Возможно, у вас установлен xlsxwriter в виртуальной среде, и вы не активируете виртуальную среду перед запуском кода.
  3. Ваша IDE использует версию Python, отличную от той, в которой есть xlsxwriter.
  4. Пакет не установлен в PyCharm

Давайте посмотрим, как исправить эти ошибки на практике.

1. У вас несколько версий Python

Если в вашей системе установлено несколько версий Python, вам необходимо убедиться, что вы используете конкретную версию, в которой доступен модуль xlsxwriter.

Вы можете проверить это, запустив команду which -a pythonили which -a python3из терминала:

$ which -a python3

/opt/homebrew/bin/python3

/usr/bin/python3



В приведенном выше примере установлены две версии Python: /opt/homebrew/bin/python3и /usr/bin/python3.

Предположим, вы выполняете следующие шаги в своем проекте:

  1. Установите xlsxwriter, pipиспользуя /usr/bin/версию Python.
  2. Установите Python с помощью Homebrew, у вас есть Python./opt/homebrew/
  3. Затем вы запускаете import xlsxwriterсвой код

Вышеуказанные шаги вызовут ошибку, поскольку xlsxwriter установлен в /usr/bin/, и ваш код, вероятно, выполняется с использованием Python из /opt/homebrew/пути.

Чтобы устранить эту ошибку, вам необходимо еще раз запустить pip install xlsxwriterкоманду, чтобы xlsxwriter был установлен и доступен для активной версии Python.

2. Виртуальная среда Python активна.

Другой сценарий, который может вызвать эту ошибку, заключается в том, что в виртуальной среде установлен xlsxwriter.

Пакет Python venvпозволяет вам создать виртуальную среду, в которой вы можете устанавливать различные версии пакетов, необходимых вашему проекту.

Если вы выполняете установку xlsxwriterвнутри виртуальной среды, модуль не будет доступен за пределами этой среды.

Вы можете увидеть, активна ли виртуальная среда или нет, посмотрев на приглашение в терминале.

Когда виртуальная среда активна, имя этой среды будет отображаться в круглых скобках, как показано ниже:

Python имеет активную среду venv

На рисунке выше отображается имя виртуальной среды (demoenv), указывающее, что виртуальная среда в данный момент активна.

Если вы запустите pip install, пока виртуальная среда активна, пакет будет установлен только для этой среды.

Аналогично, любой пакет, установленный вне этой виртуальной среды, не будет доступен из виртуальной среды. Решение заключается в запуске pip installкоманды в среде, которую вы хотите использовать.

Если вы хотите установить xlsxwriter глобально, отключите виртуальную среду, выполнив команду deactivateперед ее запуском pip install.

3. IDE с использованием другой версии Python

Наконец, среда IDE, в которой вы запускаете код Python, может использовать другую версию Python, если у вас установлено несколько версий.

Например, вы можете проверить интерпретатор Python, используемый в VSCode, открыв палитру команд ( CTRL + Shift + Pдля Windows и ⌘ + Shift + PMac), а затем запустив Python: Select Interpreterкоманду.

Вы должны увидеть все доступные версии Python, перечисленные следующим образом:

Версии Python, перечисленные в VSCode

Вам необходимо использовать ту же версию, в которой вы установили xlsxwriter, чтобы модуль можно было найти при запуске кода из VSCode.

После этого вы сможете импортировать xlsxwriter в свой код.

4. Вы видите эту ошибку в PyCharm.

Если вы используете PyCharm в качестве IDE, эта ошибка может возникнуть, поскольку пакет не установлен в интерпретаторе Python, используемом PyCharm.

Это связано с тем, что PyCharm создает новую виртуальную среду для каждого проекта, который вы создаете с помощью IDE.

Чтобы устранить эту ошибку, вы можете установить пакет с помощью терминала PyCharm.

Более подробную информацию можно найти в руководстве по установке и удалению пакетов в PyCharm .

Заключение

Таким образом, это ModuleNotFoundError: No module named 'xlsxwriter'происходит, когда xlsxwriterбиблиотека не установлена ​​в вашей среде Python. Чтобы устранить эту ошибку, вам необходимо запустить pip install xlsxwriterкоманду.

Если модуль у вас уже установлен, убедитесь, что вы используете правильную версию Python, проверьте, активна ли виртуальная среда (если она у вас есть), а также проверьте версию Python, используемую вашей IDE.

Выполнив эти шаги, вы сможете xlsxwriterуспешно импортировать модуль в свой код.
