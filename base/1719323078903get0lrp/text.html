Создание более простой и удобной таблицы дат в Power BI

Таблица дат в Power BI может иметь меньшее количество столбцов за счет использования строк настраиваемого формата для адекватного управления визуализацией диаграммы и порядком сортировки.

Реализация классической таблицы дат

Таблица дат обычно содержит такие столбцы, как «Месяц» и «День недели» , которые следует сортировать по базовому порядковому номеру, а не по алфавитному порядку названий месяцев и дней. Для этого мы создаем два столбца для этих атрибутов: один столбец с видимым именем, который отсортирован по другому столбцу, содержащему число. Например, месяц сортируется по номеру месяца в следующей вычисляемой таблице DAX:

Classic Date =

VAR FirstYear =          -- Customizes the first year to use

    YEAR ( MIN ( Sales[Order Date]  ) )

RETURN

ADDCOLUMNS (

    FILTER (

        CALENDARAUTO (),

        YEAR ( [Date] ) >= FirstYear

    ),

    "Year", YEAR ( [Date] ),

    "Year Month", FORMAT ( [Date], "mmm yyyy" ),

    "Year Month Number", YEAR ( [Date] ) * 12 + MONTH ( [Date] ),

    "Month", FORMAT ( [Date], "mmm" ),

    "Month Number", MONTH ( [Date] ),

    "Day of Week", FORMAT ( [Date], "ddd" ),

    "Day of Week Number", WEEKDAY ( [Date], 1 )

)

В предыдущем коде есть 12 имен в месяце (январь, февраль, …) и 12 имен в месяце года для каждого года (январь 2008 г., февраль 2008 г., …, декабрь 2008 г., январь 2009 г., февраль 2009 г., …).

Год Месяц полезен для создания диаграмм, подобных следующей.

Однако если линейный график содержит много точек данных на небольшой площади, внизу отображается полоса прокрутки. Действительно, поскольку ось X содержит текст, а не числа, она относится к категориальному типу. В статье Улучшение временных диаграмм в Power BI с помощью DAX мы показали, как создать дополнительные столбцы в таблице дат , чтобы установить непрерывный тип по оси X. Этот дополнительный столбец содержит первый или последний день каждого периода времени, отображаемого на линейной диаграмме. Единственным недостатком этого подхода является дополнительное количество атрибутов, необходимых в таблице дат , но результат соответствует нашим ожиданиям: все точки данных видны на линейном графике без полосы прокрутки, а ось X адаптирует плотность описания в зависимости от доступного места.

Реализация более простой таблицы дат

Благодаря недавнему внедрению строк настраиваемого формата в Power BI мы можем использовать другой подход, который больше не требует дополнительных столбцов, а также устраняет необходимость использования функции сортировки по столбцу. Основная идея заключается в том, что мы всегда сохраняем дату за такой период, как год, месяц или неделя. Строка пользовательского формата отображает текст, который мы хотим видеть в пользовательском интерфейсе; однако, поскольку базовые данные представляют собой число, мы автоматически получаем правильный порядок сортировки и возможность использовать непрерывный тип на оси X линейного графика.

Date =

VAR FirstYear =          -- Customizes the first year to use

    YEAR ( MIN ( Sales[Order Date]  ) )

RETURN

ADDCOLUMNS (

    FILTER (

        CALENDARAUTO (),

        YEAR ( [Date] ) >= FirstYear

    ),

    "Year", DATE ( YEAR ( [Date] ), 12, 31 ),

    "Year Month", EOMONTH ( [Date], 0 ),

    "Month", FORMAT ( [Date], "mmm" ),

    "Month Number", MONTH ( [Date] ),

    "Day of Week", FORMAT ( [Date], "ddd" ),

    "Day of Week Number", WEEKDAY ( [Date], 1 )

)

ОБНОВЛЕНИЕ 23 мая 2021 г. На основании полученных нами комментариев по улучшению визуализации на линейной диаграмме вы можете использовать первый день года и месяца в столбцах «Год» и «Год-месяц» — в этом случае вы можете рассмотреть возможность использования этих выражения в предыдущей формуле:


"Year", DATE ( YEAR ( [Date] ), 1, 1 ),

"Year Month", EOMONTH ( [Date], -1 ) + 1,

КОПИРОВАТЬКОНВЕНЦИИ№ 3

Без правильного формата столбцов результат столбцов «Год» и «Год-месяц» прочитать нелегко: это последний день года и месяца соответственно.

Свойство строки пользовательского формата столбцов можно изменить в представлении диаграммы. Например, этот пользовательский формат применяется к столбцу «Год-месяц» .

Мы применяем следующий формат к столбцам типа данных datetime таблицы Date :

  • Дата : м/дд/гггг (14 января 2008 г.), используется в качестве столбца для пометки таблицы дат.
  • Год : гггг (2008)
  • Год Месяц : ммм гггг (январь 2008 г.)

Мы по-прежнему используем классический подход с текстовым столбцом и числовым столбцом для порядка сортировки атрибутов «Месяц» и «День недели» . Это связано с тем, что мы вряд ли будем использовать непрерывную линейную диаграмму для этих двух атрибутов. Таблица с отформатированными столбцами выглядит следующим образом.

Несмотря на то, что синтаксис формата аналогичен тому, который поддерживается функцией ФОРМАТ , необходимо учитывать некоторые различия и ограничения. Например, не существует синтаксиса для отображения номера квартала по дате. Все подробности о поддерживаемом синтаксисе пользовательского формата можно найти в документации Microsoft.

ПРИМЕЧАНИЕ . Если вы хотите применить префикс к форматированной строке, используйте символ обратной косой черты (\) перед каждым постоянным символом, который не нужно интерпретировать как строку пользовательского формата. Например, для столбца с финансовым годом следует использовать выражение \F\Y гггг, чтобы в результате получить 2008 финансовый год .

Таблица дат имеет иерархию, хотя это не приводит к побочным эффектам на формат:

  • Календарь: Год ( Year ), Месяц ( Год Месяц ), Дата ( Дата )

Используя новую таблицу дат , мы получаем следующие матрицы, в которых столбцы «Год-месяц» и «Месяц» отображают желаемый текст вместо базовых дат.

Еще одним преимуществом этого подхода является то, что вам не нужно беспокоиться о дополнительном столбце, используемом при сортировке по столбцу, если вы манипулируете контекстом фильтра для расчета DAX. Подробнее об этом можно прочитать в статье « Побочные эффекты настройки сортировки по столбцу в DAX» .

Когда мы перемещаемся по столбцу «Год-месяц» с помощью линейной диаграммы, мы получаем желаемый результат.

Ограничения и дополнительная информация

Использование типа данных даты для столбцов, представляющих год и месяц (включая год), — это удобный способ управлять форматом отображения и продолжать использовать категориальный тип на оси X линейной диаграммы. К сожалению, мы не можем использовать этот метод для четвертей, поскольку синтаксис пользовательского формата не поддерживает четверти. Таким образом, мы должны использовать классический подход с двумя столбцами для квартального уровня календарной иерархии: один столбец содержит строку (например, Q1-2008), а другой столбец содержит порядок сортировки. Вы можете избежать дополнительного столбца сортировки, только если используете соглашение об именовании, которое автоматически обеспечивает правильный порядок сортировки, например 2008-Q1 вместо Q1-2008.

Когда вы используете срез с условием «между» в столбце типа «Год» и «Год-месяц» , определенном с типом данных datetime, срез всегда показывает все возможные даты, даже если базовая степень детализации осуществляется по году или месяцу. Выбор будет включать весь месяц или год только в том случае, если соответствующая дата находится в пределах диапазона, но визуализация может ввести пользователя отчета в заблуждение.

В предыдущем примере мы продолжали использовать подход с двумя столбцами для атрибутов «Месяц» и «День недели» . Эти столбцы содержат только 12 и 7 значений соответственно, независимо от количества лет в таблице дат . Таким образом, невозможно определить соответствующую дату, поскольку январь объединяет даты января всех лет, включенных в таблицу дат .

Однако можно назначить произвольную дату для всех дат января и другую дату для всех дат понедельника. При правильной настройке строки пользовательского формата пользователь не сможет увидеть разницу, а для модели требуется только один столбец вместо двух столбцов для каждого из этих атрибутов. Хотя это может упростить манипулирование контекстом фильтра и может считаться более «элегантным» решением, на самом деле это не дает реального преимущества конечным пользователям. По умолчанию они увидят произвольную дату, если к линейному графику будет применен один из атрибутов «Месяц» или «День недели». В этом случае тип оси X следует изменить с непрерывного на категориальный, чтобы восстановить правильную визуализацию. Поскольку целью статьи было упростить взаимодействие с пользователем отчета, мы не использовали этот подход в предлагаемом решении. Если вы хотите попробовать этот подход, вы можете попробовать таблицу экстремальных дат в образце файла, которая назначает месяц и день недели произвольным датам в 1900 году.

Автоматическое поведение даты/времени

Другой обходной путь использования непрерывной оси — использование тех же метаданных, которые используются функцией автоматической даты и времени в Power BI. Таблица автоматических дат отображает единую иерархию, в которой используется непрерывная ось, хотя столбцы «Квартал» и «Месяц» являются категориальными. Это можно получить, используя определенные атрибуты таблицы дат:

  • Таблица дат — свойство DataCategory: Время (автоматически устанавливается с помощью параметра «Пометить как таблицу дат»).
  • Столбец даты — свойство DataCategory: PaddedDateTableDates (необходимо установить с помощью табличного редактора)
  • Столбец года – свойство DataCategory: Годы (необходимо установить с помощью табличного редактора).

Более того, навигация должна осуществляться с использованием иерархии: простое размещение отдельных атрибутов на оси не включает тип оси Непрерывный для диаграмм в Power BI.

Выводы

Более простая таблица дат может иметь меньшее количество столбцов, если вы используете строку пользовательского формата. Удаление требования сортировки по столбцу также упрощает код DAX, если вы манипулируете контекстом фильтра. Существует только одно предупреждение, связанное с небольшими различиями между синтаксисом формата, используемым Power BI, и функцией FORMAT в DAX. Используйте синтаксис поддерживаемого пользовательского формата в документации Microsoft в качестве справочника по Power BI.
