Это работает для всех:

pip install -r /path/to/requirements.txt


Объяснение:

-r, --requirement <имя файла>

Установите из данного файла требований. Эту опцию можно использовать несколько раз.



Это работает для меня:

$ pip install -r requirements.txt --no-index --find-links file:///tmp/packages


--no-index- Игнорировать индекс пакета (вместо этого смотреть только --find-linksURL-адреса).

-f, --find-links <URL>- Если <URL>это URL-адрес или путь к HTML-файлу, то проанализировать ссылки на архивы. Если <URL>это локальный путь или file://URL-адрес каталога, найдите архивы в списке каталогов.

Короткий ответ

pip install -r /path/to/requirements.txt


или в другой форме:

python -m pip install -r /path/to/requirements.txt


Объяснение

Вот -rкраткая форма --requirementи просит pipустановить из данного requirementsфайла.

pipначнет установку только после проверки доступности всех перечисленных элементов в requirementsфайле и не начнет установку, даже если один из них requirementнедоступен.

Одним из обходных путей установки доступных пакетов является установка перечисленных пакетов один за другим. Используйте для этого следующую команду. Будет показано предупреждение красного цвета, уведомляющее вас о недоступных пакетах.

cat requirements.txt | xargs -n 1 pip install


Чтобы игнорировать комментарии (строки, начинающиеся с a #) и пустые строки, используйте:

cat requirements.txt | cut -f1 -d"#" | sed '/^\s*$/d' | xargs -n 1 pip install

Часто вам потребуется быстрая установка из локальных архивов без проверки PyPI.

Сначала скачайте архивы, соответствующие вашим требованиям:

$ pip install --download <DIR> -r requirements.txt


Затем установите с помощью –find-linksи –no-index:

$ pip install --no-index --find-links=[file://]<DIR> -r requirements.txt
