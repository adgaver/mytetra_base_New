Введение

Каталог пользовательских данных содержит данные профиля, такие как история, закладки и файлы cookie, а также другие локальные данные для каждой установки.

Каждый профиль представляет собой подкаталог (часто Default) внутри каталога пользовательских данных.

Текущее местоположение

Чтобы определить каталог пользовательских данных для работающего экземпляра Chrome:

  1. Перейдите кchrome://version
  2. Ищите Profile Pathполе. Это дает путь к каталогу профиля.
  3. Каталог пользовательских данных является родительским для каталога профиля.

Пример (Windows):

  • [Путь к профилю]C:\Users\Alice\AppData\Local\Google\Chrome\User Data\Default
  • [Каталог пользовательских данных]C:\Users\Alice\AppData\Local\Google\Chrome\User Data

Местоположение по умолчанию

Местоположение каталога пользовательских данных по умолчанию вычисляется с помощью chrome::GetDefaultUserDataDirectory.

Обычно это зависит от

  • платформа ОС,
  • брендинг ( Chrome против Chromium , на основе is_chrome_brandedаргументов GN ) и
  • канал выпуска (stable/beta/dev/canary).

Окна

Местоположение по умолчанию находится в локальной папке данных приложения:

  • [Хром]%LOCALAPPDATA%\Google\Chrome\User Data
  • [Хром Бета]%LOCALAPPDATA%\Google\Chrome Beta\User Data
  • [Хром Канарейка]%LOCALAPPDATA%\Google\Chrome SxS\User Data
  • [Хром]%LOCALAPPDATA%\Chromium\User Data

(Суффикс канареечного канала определяется с помощью InstallConstants::install_suffix.)

Mac OS X

Местоположение по умолчанию находится в Application Supportпапке:

  • [Хром]~/Library/Application Support/Google/Chrome
  • [Хром Бета]~/Library/Application Support/Google/Chrome Beta
  • [Хром Канарейка]~/Library/Application Support/Google/Chrome Canary
  • [Хром]~/Library/Application Support/Chromium

(Суффикс канареечного канала определяется с помощью CrProductDirNameключа в браузерном приложении Info.plist.)

Линукс

Местоположение по умолчанию ~/.config:

  • [Хром Стабильный]~/.config/google-chrome
  • [Хром Бета]~/.config/google-chrome-beta
  • [Разработчик Chrome]~/.config/google-chrome-unstable
  • [Хром]~/.config/chromium

(Суффиксы каналов бета-версии и разработки определяются из $CHROME_VERSION_EXTRA, который передается сценарием-оболочкой запуска .)

Часть ~/.configместоположения по умолчанию может быть переопределена с помощью $CHROME_CONFIG_HOME(начиная с M61) или с помощью $XDG_CONFIG_HOME.

Обратите внимание, что это $XDG_CONFIG_HOMEкасается всех приложений, соответствующих спецификации базового каталога XDG , но $CHROME_CONFIG_HOMEотносится только к Chrome и Chromium.

Хром ОС

Местоположение по умолчанию:/home/chronos

Андроид

Местоположение по умолчанию берется из Context.getDir и зависит от приложения.

Пример:/data/user/0/com.android.chrome/app_chrome

iOS

Местоположение по умолчанию — внутри каталога поддержки приложений в изолированной программной среде приложения.

  • [Хром]Library/Application Support/Google/Chrome
  • [Хром]Library/Application Support/Chromium

Переопределение каталога пользовательских данных

Командная строка

На большинстве платформ каталог пользовательских данных можно переопределить, передав --user-data-dirфлаг командной строки в двоичный файл Chrome.

Переопределение происходит в формате chrome/app/chrome_main_delegate.cc. Платформы, не использующие этот файл, возможно, не реализовали переопределение. Переопределение каталога пользовательских данных через командную строку не поддерживается в iOS.

Пример:

  • [Окна]chrome.exe --user-data-dir=c:\foo
  • [Линукс]google-chrome --user-data-dir=/path/to/foo

Окружающая среда (Linux)

В Linux каталог пользовательских данных также можно переопределить с помощью $CHROME_USER_DATA_DIRпеременной среды.

Флаг --user-data-dirимеет приоритет, если присутствуют оба.

Сеансы удаленного рабочего стола Chrome (Linux)

Удаленный рабочий стол Chrome (CRD) используется для установки $CHROME_USER_DATA_DIRвиртуального $CHROME_CONFIG_HOMEсеанса на хосте Linux, поскольку один экземпляр Chrome не может отображать окна на нескольких дисплеях X, а два запущенных экземпляра Chrome не могут использовать один и тот же каталог пользовательских данных. Однако с устареванием Linux dbus-x11большинство современных дистрибутивов Linux утратили возможность одновременно запускать несколько графических сеансов для одного и того же пользователя, не сталкиваясь при этом с трудно отслеживаемыми проблемами перекрестных помех в dbus, а Chrome можно запускать только на одном X-дисплее каждый раз. пользователь в реальности. Таким образом, CRD больше не устанавливает эти переменные среды для новых установок после узла CRD M105.

Хост CRD продолжит устанавливать эти переменные среды, если они chrome-config/существуют chrome-profile/в ~/.config/chrome-remote-desktop/. Если вы хотите использовать локальный профиль Chrome в сеансах CRD, закройте Chrome и удалите эти папки из ~/.config/chrome-remote-desktop/, а затем перезагрузите хост-устройство.

Написание оболочки AppleScript (Mac OS X)

В Mac OS X вы можете создать приложение, запускающее Chrome, со специальной --user-data-dir:

  1. Откройте «Приложения» > «Утилиты» > «Редактор сценариев».

  2. Входить:

установите для Chrome значение "\"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome\""

установите для userdatadir значение «\"$HOME/Library/Application Support/Google/Chrome Alt\""

выполнить сценарий оболочки chrome & " --user-data-dir=" & userdatadir & " > /dev/null 2>&1 &"



  1. При необходимости измените путь установки Chrome или Chromium и желаемый каталог пользовательских данных.

  2. Сохраните сценарий в каталоге «Приложения» в формате файла «Приложение».

  3. Закройте редактор сценариев, найдите только что созданное приложение и запустите его. Откроется экземпляр Chrome, указывающий на ваш новый профиль.

Если хотите, можете присвоить этому приложению тот же значок, что и Chrome:

  1. Выберите приложение Google Chrome и выберите «Файл» > «Получить информацию».
  2. Выберите значок в левом верхнем углу информационного диалогового окна. Вокруг значка вы увидите синюю подсветку.
  3. Нажмите ⌘C, чтобы скопировать значок.
  4. Откройте диалоговое окно информации о новом приложении и выберите значок в левом верхнем углу.
  5. Нажмите ⌘V, чтобы вставить скопированный значок.

Каталог пользовательского кэша

В Windows и ChromeOS каталог пользовательского кэша совпадает с каталогом профиля. (Каталог профиля находится внутри каталога пользовательских данных.)

В Mac OS X и iOS каталог пользовательского кэша получается из каталога профиля следующим образом:

  1. Если Library/Application Supportэто предок каталога профиля, каталог пользовательского кэша Library/Cachesплюс относительный путь от Application Supportкаталога профиля.
  2. В противном случае каталог пользовательского кэша совпадает с каталогом профиля.

Пример (Mac OS X):

  • [каталог пользовательских данных]~/Library/Application Support/Google/Chrome
  • [каталог профиля]~/Library/Application Support/Google/Chrome/Default
  • [каталог кэша пользователя]~/Library/Caches/Google/Chrome/Default

В Linux каталог пользовательского кэша получается из каталога профиля следующим образом:

  1. Определите каталог конфигурации системы. Это так ~/.config, если только это не переопределено $XDG_CONFIG_HOME. (Этот шаг игнорирует $CHROME_CONFIG_HOME.)
  2. Определите каталог системного кэша. Это так ~/.cache, если только это не переопределено $XDG_CACHE_HOME.
  3. Если каталог конфигурации системы является предком каталога профиля, каталог пользовательского кэша — это каталог системного кэша плюс относительный путь от каталога конфигурации системы до каталога профиля.
  4. В противном случае каталог пользовательского кэша совпадает с каталогом профиля.

Пример (Linux):

  • [каталог пользовательских данных]~/.config/google-chrome
  • [каталог профиля]~/.config/google-chrome/Default
  • [каталог кэша пользователя]~/.cache/google-chrome/Default

В Android каталог пользовательского кэша находится в Context.getCacheDir .
