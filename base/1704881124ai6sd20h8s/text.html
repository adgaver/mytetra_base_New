Удалить лишние пробелы между словами в Power Query непросто. Несмотря на встроенную функцию Power Query для управления начальными и конечными пробелами, удаление повторяющихся пробелов в строке может оказаться сложной задачей.

В этой статье будут продемонстрированы три эффективных метода изящного решения этой проблемы. Мы углубимся в стратегии от создания рекурсивных функций с оператором @ до использования List.Generate для итеративных замен. Мы также рассмотрим уникальный подход к разделению и объединению текстовых значений. К концу этого руководства вы получите четкое представление о том, как заменить любую серию последовательных пробелов одним в Power Query.

Удаление пробелов

Наша цель проста: заменить все последовательные пробелы одним. Существует несколько подходов к этому, и в этой статье рассматриваются три.

Рекурсия с использованием оператора @

При работе с последовательными пробелами один из подходов предполагает использование рекурсии в сочетании с оператором @. Рекурсия позволяет функции вызывать саму себя, что делает ее подходящим методом для итеративных операций. Включив оператор @, мы можем добиться эффективной и лаконичной рекурсии в Power Query.

Для начала мы начнем с базового оператора, который заменяет двойные пробелы одинарными пробелами в текстовом значении:

Начнем с основных утверждений, таких как:

Text.Replace( "G o r ill a", " ", " " ) // Returns "G o r ill a"


КОПИРОВАТЬ

Далее мы улучшаем эту операцию, делая ее рекурсивной, гарантируя, что замена двойных пробелов будет продолжаться до тех пор, пока не будет достигнут желаемый результат. Это достигается за счет использования оператора «if» в нашей рекурсивной функции:

fxRemoveSpaces = ( x as text ) as text =>

if Text.Contains( x, " " )

then Text.Replace( x, " ", " " )

else x


КОПИРОВАТЬ

Чтобы сделать эту функцию рекурсивной, мы используем оператор @ для вызова самой себя:

fxRemoveSpaces = ( x as text ) as text =>

if Text.Contains( x, " " )

then @fxRemoveSpaces( Text.Replace( x, " ", " " ) )

else x


КОПИРОВАТЬ

В приведенном выше примере @fxRemoveSpaces рекурсивно вызывает саму функцию. Применяя эту функцию к текстовому значению, мы достигаем желаемого результата — удаления последовательных пробелов.

В этом случае @fxRemoveSpaces рекурсивно вызывает саму функцию. Применяя эту функцию к примеру, мы получаем:

// Below example returns "G o r ill a "

let

fxRemoveSpaces = ( x as text ) as text =>

if Text.Contains( x, " " )

then @fxRemoveSpaces( Text.Replace( x, " ", " " ) )

else x,

result = fxRemoveSpaces( "G o r ill a " )

in

result


КОПИРОВАТЬ

Чтобы применить рекурсию и оператор @ к настраиваемому столбцу, например «[MyText], мы можем использовать функцию Table.AddColumn :

Table.AddColumn(

Source,

"CleanText",

each

let

fxRemoveSpaces = ( x as text ) as text =>

if Text.Contains( x, " " )

then @fxRemoveSpaces( Text.Replace( x, " ", " " ) )

else x,

result = fxRemoveSpaces( [MyText] )

in

result,

type text

)


КОПИРОВАТЬ

Включив рекурсивную функцию в функцию Table.AddColumn , мы создаем новый столбец «CleanText», который содержит текстовое значение после удаления последовательных пробелов.

Соображения: Важно отметить, что рекурсия с оператором @ иногда может потреблять значительный объем памяти и может быть не самым эффективным методом для больших наборов данных. В качестве альтернативы в следующем разделе мы рассмотрим другие подходы для оптимизации производительности и распределения памяти.

Итерация с помощью List.Generate

Когда дело доходит до решения проблемы удаления последовательных пробелов, альтернативный метод предполагает использование возможностей List.Generate . Эта универсальная функция позволяет выполнять итеративные действия на основе заданных условий. Используя List.Generate , вы можете перебирать текстовое значение, проверяя наличие нескольких пробелов и заменяя их по мере необходимости.

Чтобы проиллюстрировать применение List.Generate , давайте рассмотрим текстовое значение «G или ill a», которое содержит несколько последовательных пробелов. Реализуя List.Generate , мы можем постепенно заменять несколько пробелов.

/* Returns three list items, each with another replacement.

{

"G o r ill a",

"G o r ill a",

"G o r ill a"

}

*/

List.Generate(

() => "G o r ill a",

each Text.Contains( _, " " ),

each Text.Replace( _, " ", " " )

)


КОПИРОВАТЬ

Функция List.Generate начинается с исходного текстового значения, а затем проверяет, содержит ли оно последовательные пробелы, с помощью функции Text.Contains . Затем он итеративно заменяет последовательные пробелы одним пробелом с помощью функции Text.Replace . Выходные данные представляют собой список значений, каждое из которых представляет текст после итерации замены.

Хотя список замен имеет ценность, наша основная задача — получить окончательную чистую строку. Мы можем извлечь последнее значение из списка. Однако в строке все еще присутствует оставшийся двойной пробел. Поэтому, чтобы убедиться, что последняя замена выполнена правильно, мы можем использовать необязательный четвертый аргумент, чтобы обеспечить дополнительную замену в селекторе.

// Returns "G o r ill a"

let

Replacements =

List.Generate(

() => "G o r ill a",

each Text.Contains( _, " " ),

each Text.Replace( _, " ", " " ),

each Text.Replace( _, " ", " " )

),

Result = List.Last( Replacements )

in

Result



КОПИРОВАТЬ

Добавив эту дополнительную замену и используя функцию List.Last , мы получаем окончательную чистую строку, гарантирующую удаление последовательных пробелов.

Чтобы избежать ненужного повторения кода, мы можем преобразовать действие замены в функцию. Такой подход позволяет нам дважды вызывать функцию внутри List.Generate , сокращать код и улучшать удобство сопровождения.

// Returns "G o r ill a"

let

Replacer = (t) => Text.Replace( t, " ", " " ),

Replacements =

List.Generate(

() => "G o r ill a",

each Text.Contains( _, " " ),

Replacer,

Replacer

),

Result = List.Last( Replacements )

in

Result



КОПИРОВАТЬ

Зафиксировав логику замены в функции replacer, мы достигаем более краткого и элегантного решения, сохраняя при этом удаление всех последовательных пробелов.

Чтобы дополнительно изучить возможности List.Generate, обязательно прочитайте эту подробную статью о List.Generate .

Итерация с List.Accumulate

В нашем стремлении удалить последовательные пробелы мы можем использовать функцию List.Accumulate , которая особенно полезна, когда количество итераций заранее определено. Хотя при работе со строками точное количество требуемых итераций может быть неизвестно, мы можем сделать предположение, обеспечивающее эффективную обработку.

Чтобы решить проблему последовательных пробелов в строке, важно оценить максимальное количество последовательных пробелов, которые могут возникнуть. Учитывая общую длину строки, мы можем определить приблизительный максимум. Эффективный подход — разделить длину на 2 с помощью функции Number.IntegerDivide . Это деление даст целую часть результата, служащую оценкой максимального количества последовательных пробелов.

Давайте рассмотрим практический пример, чтобы проиллюстрировать использование List.Accumulate . Предположим, у нас есть строка «G или ill a», содержащая несколько последовательных пробелов. Мы можем использовать List.Accumulate , чтобы удалить лишние пробелы и получить чистую версию строки.

let

Source = "G o r ill a",

Result =

List.Accumulate(

List.Repeat( { " " }, Number.IntegerDivide (Text.Length( Source ), 2 ) ),

Source,

( state, current ) => Text.Replace( state, current, " " )

)

in

Result


КОПИРОВАТЬ

Функция List.Accumulate перебирает список повторяющихся последовательных пробелов (на основе нашего предположения) и постепенно заменяет их одним пробелом с помощью функции Text.Replace . Конечным результатом будет правильно отформатированная строка без лишних пробелов.

Для дальнейшего изучения возможностей List.Accumulate обратитесь к подробной статье List.Accumulate .

Разделение и объединение текстовых значений

В отличие от предыдущих подходов, этот метод использует другой путь. Вместо того, чтобы заменять последовательные пробелы, мы удаляем все пробелы и вводим их снова позже в процессе.

Первый шаг — разделить текстовое значение по каждому вхождению пробела. Для этого вы можете использовать функцию Text.Split .

Text.Split( "G o r ill a", " " )


КОПИРОВАТЬ

Приведенный выше код вернет список с отдельными элементами, представляющими разделенные части текстового значения. Например, результат может быть {"G", "", "o", "", "", "", "", "", "r", "", "ill", "", "", "a"}.

Чтобы удалить пустые строки (представляющие последовательные пробелы) из списка, вы можете использовать функцию List.RemoveItems .

let

TextSplitBySpace = Text.Split( "G o r ill a", " " ),

ListWithoutEmpty = List.RemoveItems( TextSplitBySpace, {""} ),

in

ListWithoutEmpty


КОПИРОВАТЬ

Приведенный выше фрагмент кода удаляет пустые строки из списка, в результате чего получается {"G", "o", "r", "ill", "a"}. Теперь у нас есть список, содержащий отдельные слова или элементы исходного текстового значения.

Поскольку между остальными элементами списка изначально было хотя бы одно пространство между собой, теперь мы можем снова ввести пробелы с помощью функции Text.Combine .

let

// Split string into words

TextSplitBySpace = Text.Split( "G o r ill a", " " ),

// Remove empty strings (consecutive spaces)

ListWithoutEmpty = List.RemoveItems( TextSplitBySpace, {""} ),

// Combine words with single space

TextCombined = Text.Combine( ListWithoutEmpty, " " )

in

TextCombined


КОПИРОВАТЬ

Конечным результатом приведенного выше фрагмента кода является «G или ill a», где последовательные пробелы удалены, а между каждым словом вставлен один пробел.

Заключение

Очистка лишних пробелов в тексте может оказаться непростой задачей, но, как вы видели, в Power Query есть разные способы справиться с ними.

Во-первых, мы увидели, как оператор @ может неоднократно вызывать себя, пока задача не будет выполнена. Далее мы проверили, как List.Generate и List.Accumulate выполняют итерацию по списку значений. И, наконец, мы разделяем значения на части, а затем собираем их вместе.

С помощью этих удобных методов вы теперь готовы справиться с любыми дополнительными пробелами, которые могут появиться в ваших текстовых данных.
