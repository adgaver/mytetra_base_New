Основы

Замена значений — полезный навык, который стоит освоить. Прежде чем перейти к более сложным темам, мы начнем с основ.

Как заменить значения?

В самой простой форме замена значений в Power Query проста. Если вы хотите заменить значения в столбце, вы можете:

1. Щелкните правой кнопкой мыши столбец -> выберите «Заменить значения».

2. Перейдите на вкладку «Преобразование» -> нажмите «Заменить значения».

3. Вы можете щелкнуть правой кнопкой мыши значение в столбце и выбрать «Заменить значения».

После любого из трех шагов появится всплывающее окно «Заменить значения».

Аргументы Table.ReplaceValue

Теперь давайте посмотрим, как работает замена значения. В столбце «Семейное_статус» вы хотите заменить текстовое значение «Вместе» на слова « Живем вместе» . Для этого на экране замены значений введите следующее:

  • Ценность найти: Вместе
  • Заменить на: Жить вместе.

После нажатия кнопки «ОК» Power Query выполнит замену. Если вы заменили значение впервые, поздравляем! Как видите, это очень легко.

Давайте подробнее рассмотрим код, который генерирует операция замены. Функция Table.ReplaceValue выполняет замену и требует 5 аргументов. Ниже вы найдете код замены, которую мы только что сделали:

После форматирования кода для наглядности вы можете найти аргументы функции ниже:

= Table.ReplaceValue(

#"Changed Type",

"Together",

"Living Together",

Replacer.ReplaceText,

{"Marital_Status"}

)


5 аргументов внутри функции:

Заменитьзначение против Заменитьтекст

Существует разница между заменой текстовых значений и заменой других типов данных . В предыдущем примере мы заменили текстовое значение, используя:

= Table.ReplaceValue(

#"Changed Type",

"Together",

"Living Together",

Replacer.ReplaceText,

{"Marital_Status"}

)


Этот код генерируется автоматически при замене значений в текстовом столбце. Вы также можете заменить числовые значения, например год рождения . Если изменить год с 1957 на 1958, код будет выглядеть так:

= Table.ReplaceValue(

#"Replaced Value",

1957,

1958,

Replacer.ReplaceValue,

{"Year_Birth"}

)


Разница между двумя примерами заключается в четвертом аргументе — функции replacer . Аргумент может вызывать одну из двух функций:

  1. Заменитель.ReplaceText
  2. Заменить.ReplaceValue

Для замены значений можно использовать функции Table.ReplaceValue и List.ReplaceValue . И хотя они оба заменяют элементы, их поведение различно. Версия replacer.ReplaceText также может заменять подстроки . Тем не менее , replacer.ReplaceValue заменяет только содержимое ячейки целиком .

Что это значит? Представьте, что вы хотите заменить текст «Выпускной» на «Бакалавриат». Когда все содержимое ячейки совпадает, замена будет успешной, как показано на рисунке ниже.

Тем не менее, когда вы пытаетесь заменить подстроку строки, замена не удалась.

Изменение функции замены на текстовую версию позволяет заменить частичный текст.

Всплывающее окно замены значений для текстовых значений выглядит, как показано ниже. Можете ли вы предсказать, что произойдет, если вы установите флажок «Сопоставлять все содержимое ячейки» в разделе « Дополнительные параметры» ? Затем Power Query использует функцию replacer.ReplaceValues . Тогда как непроверенная версия использует версию replaceText.

Таким образом, функция replacer.ReplaceValue заменяет текст, когда все содержимое ячейки совпадает. Наоборот не работает. Применение функции replacer.ReplaceText к числовому столбцу приводит к ошибке: « Expression.Error: мы не можем преобразовать значение 1965 в тип Text ».

Заменить в нескольких столбцах

До сих пор вы заменили значения в одном столбце. Но что, если вы хотите заменить более одного столбца? Без проблем. Для этого вы можете выбрать несколько столбцов, удерживая клавишу Ctrl и щелкая по каждому нужному столбцу. Выбрав столбцы, щелкните правой кнопкой мыши имя столбца и нажмите «Заменить значения» .

Если все выбранные вами столбцы являются текстовыми, код, сгенерированный для замен, выглядит следующим образом:

= Table.ReplaceValue(

#"Replaced Value2",

"a",

"b",

Replacer.ReplaceText,

{"ID", "Year_Birth", "Education"}

)


Обратите внимание, что в строке 6 теперь отображаются все имена столбцов, которые вы выбрали для операции замены, в списке. Таким образом, ключом к применению простой операции замены для нескольких столбцов является включение столбцов в список в пятом аргументе Table.ReplaceValue .

И все же есть на что обратить внимание. Если один из типов столбцов отличается от типа текста, ваш код будет включать функцию замены replacer.ReplaceValue .

= Table.ReplaceValue(

#"Changed Type",

"a",

"b",

Replacer.ReplaceValue,

{"ID", "Year_Birth", "Education"}

)


Когда это произойдет, знайте, что при этом текстовые значения заменяются только на точное совпадение. Другими словами, сопоставление всего содержимого ячейки. Если это нежелательное поведение, вы можете:

  1. измените типы данных всех выбранных столбцов на текст
  2. выполнить две отдельные операции замены: одну для текста и одну для других типов данных.

Условная замена: один столбец

Часто недостаточно заменить значения столбцов. Вы можете захотеть заменить значения в столбце на основе условия, и только если условие истинно, вы заменяете значения. Длинный путь к этому заключается в следующем:

  1. добавить пользовательский/условный столбец
  2. создайте условие if и верните правильные значения
  3. удалить старый столбец
  4. переименуйте недавно добавленный столбец в нужное имя

Это много шагов для условной замены значения. Итак, как можно проще заменить значения условием if? В наборе данных есть столбец, содержащий доход наших клиентов. Вы хотите максимизировать доходы в наборе данных и хотите получать зарплату только до 50 000 евро. Суммы с более высоким доходом должны быть указаны как 50 000 евро. Для этого вы сначала подготавливаете свой код.

Вместо того, чтобы писать это с нуля, вы можете создать шаблон следующим образом:

  • Щелкните правой кнопкой мыши столбец дохода -> нажмите «Заменить значения».
  • Введите случайное число в поле «Значение для поиска» и значение для замены.
  • Нажмите ОК .

Это создает следующую настройку:

= Table.ReplaceValue(

#"Changed Type",

5,

6,

Replacer.ReplaceValue,

{"Income"}

)


Вы можете использовать это как шаблон для начала. В приведенном выше коде показано значение для поиска (= 5) в аргументе 2 и замена на в аргументе 3 (= 6). Оба значения жестко запрограммированы. Когда вы условно заменяете значения, вам нужно переписать эти два аргумента.

Вместо жесткого кодирования искомого значения вы можете написать:

= Table.ReplaceValue(

#"Changed Type",

each [Income],

each if [Income] > 50000 then 50000 else [Income] ,

Replacer.ReplaceValue,{"Income"}

)


Каждый [Income] указывает функции Table.ReplaceValue заменить каждое значение из столбца Income . Приятно осознавать, что это то же имя столбца, которое вы видите в аргументе 5.

Следующий аргумент содержит условие if, которое необходимо проверить перед заменой значений: каждый if [Income] > 50000, затем 50000 else [Income] . Слово «каждое» указывает, что функция применяет логику для всех строк индивидуально.

В других случаях вы можете захотеть условно заменить один столбец значениями другого столбца. Для некоторых это может быть очевидно, но приятно осознавать, что вы можете сделать это с помощью того же кода. В следующем примере возвращается год рождения клиента , если его доход ниже 50 000 евро. В примере показано, что вы можете изменить один столбец на основе другого столбца.

Вы можете добиться того же результата с помощью следующего, еще более простого кода, предложенного StunDexim:

= Table.TransformColumns(

#"Remove nulls",

{"Income", each if _ > 50000 then 50000 else _ }

)


Условная замена: несколько столбцов

Могут быть случаи, когда вы захотите применить одно и то же преобразование к нескольким столбцам. Как мы видели ранее, выполнить обычную операцию замены нескольких столбцов очень просто. Вы добавляете столбцы в список столбцов для поиска. Но что, если вы хотите условно заменить значения нескольких столбцов за один шаг, возможно ли это?

Ограниченная версия

В некоторой степени это возможно с более ранними формулами, но имеет большое ограничение. Давайте посмотрим пример. Внимательно посмотрите на приведенный ниже набор данных. Вы попытаетесь условно заменить значения столбцов KidHome и Teenhome . Обратите внимание, что из 11 строк значения 4 строк идентичны.

Когда мы заменяем значения в нескольких столбцах, первый вопрос: какие значения мы заменяем? Допустим, мы заменяем значения из столбца Kidhome на значение 2. И вместо ввода только KidHome в качестве столбца для поиска вы также вводите Teenhome в строке 5.

= Table.ReplaceValue(

#"Removed Other Columns",

each [Kidhome],

each 2,

Replacer.ReplaceValue,

{"Kidhome", "Teenhome"}

)


При этом отмеченные значения заменяются на цифру 2. При переборе значений столбца каждая строка проверяется на равенство со значениями Kidhome. Это по определению заменяет все значения из столбца Kidhome и заменяет значения столбца Teenhome только для тех, которые соответствуют той же строке в столбце Kidhome.

Чтобы получить точные результаты, функция должна знать контекст соответствующего столбца. Таким образом, он будет знать, что ему следует заменить значения в текущем столбце . К сожалению, Power Query не знает концепции контекста столбца. Вот почему этот метод оказывается затруднительным при применении условной логики для замен в нескольких столбцах. Так как же лучше подойти к этому?

За один шаг

Поскольку предыдущий метод не увенчался успехом, мы перейдем к совершенно другому подходу. Этот подход заменяет искомое значение и функцию замены .

= Table.ReplaceValue(

#"Removed Other Columns",

each [Income] > 50000,

each 2,

(x,y,z)=> if y then z else x,

{"Kidhome", "Teenhome"}

)


Ziying35 показал этот подход на форуме. Я еще не до конца понимаю внутреннюю работу (x,y,z)=> if y then z else x . Что меня удивляет, так это то, что он не использует ни одну из функций замены из документации. Если кто-то может пролить свет на его работу, пожалуйста, дайте мне знать в комментариях. В противном случае я не хотел бы скрывать от вас этот метод. Формула условно заменяет значения в нескольких столбцах за один шаг.

Использование Unpivot

Третий подход предполагает изменение формы ваших данных. При работе с условными заменами в нескольких столбцах вам поможет операция разворота . Вы можете отменить поворот столбцов, требующих замены. Но сначала обратите внимание, что в наборе данных на изображении выше нет уникального идентификатора. Если мы отменим поворот, не имея уникального столбца, у нас возникнут проблемы с повторным поворотом данных позже. Чтобы решить эту проблему:

  • перейдите в «Добавить столбец» -> нажмите «Индексный столбец»
  • теперь выберите столбцы Kidhome и Teenhome , щелкните правой кнопкой мыши имя столбца -> выберите Unpivot Columns.

Отсюда вы можете заменить элементы в столбце «Значение» , используя метод из предыдущей главы. Следующий код заменит каждый предмет, принадлежащий клиентам с доходом менее 50 000 евро, на 2 и в противном случае вернет исходное значение.

= Table.ReplaceValue(

#"Unpivoted Other Columns",

each [Value],

each if [Income] >= 50000 then 2 else [Value],

Replacer.ReplaceValue,{"Value"}

)


Единственное, что осталось сделать, это вернуть данные в исходную форму:

  • Выберите столбец «Атрибут» -> «Перейти к преобразованию» и нажмите «Сводный столбец» .
  • Убедитесь, что в столбце «Значения» установлено значение «Значение» .
  • И установите для функции совокупного значения значение «Не агрегировать».

Если порядок сортировки важен, теперь вы можете отсортировать данные по индексному столбцу и завершить упражнение, удалив индексный столбец. Теперь вы условно заменили значения в нескольких столбцах, хотя и с обходным путем.

Так стоит ли того дополнительных шагов? Применяя свою логику к большому количеству столбцов, это облегчит вам жизнь. Кроме того, добавление столбца индекса не всегда требуется. Если ваши данные содержат уникальный идентификатор, вы можете пропустить этот шаг. В конце концов, вы сами решаете, стоят ли эти шаги затраченных усилий. Но, по крайней мере, теперь вы знаете другой способ сделать это.

Массовая замена с использованием таблицы перевода

До сих пор замены включали одно значение, которое нужно было найти , и одно значение, которым можно было заменить . Рано или поздно вам захочется заменить несколько элементов сразу, каждый с разным результатом. Представьте себе замену 20 значений, это потребует большого количества шагов в вашем запросе. Вместо добавления отдельных шагов для каждой замены вы теперь узнаете, как можно выполнить пакетную замену значений за один шаг.

Для начала вам понадобится таблица, содержащая значения замены. Мы будем использовать следующую таблицу с именем TranslationTable . Он содержит значения, которые вы хотите заменить (старые), и значения, которые вы хотите вернуть (новые). Эта операция заменяет значения только в том случае, если столбец содержит ключевое слово в столбце Old.

Представьте, что вы хотите заменить значения в приведенных ниже данных новыми значениями в таблице перевода.

Для этой операции мы будем использовать функцию Text.Replace . Эта функция имеет три аргумента, которые должны быть текстовыми значениями:

  • Текст для замены
  • Старое значение
  • Новое значение

Простую замену букв «рилла» можно выполнить, добавив в специальный столбец следующее:

= Text.Replace( [Consumer_ID], "rilla", "RICKK" )


Замена текста учитывает регистр и заменяет подстроки. Полезно знать, что функция Text.Replace выполняет только одну замену за раз. Для пакетной замены вам необходимо вызвать функцию несколько раз, используя другую функцию. Кандидатами для них являются List.Accum u late и List.Generate . В этом примере вы будете использовать версию List.Accumulate , предоставленную Гилом Равивом.

Функция List.Accumulate получает текст [Consumer_ID] и заменяет [Старый] текст соответствующим текстом [Новый] . Функция, которая выполняет замену, следующая:

=List.Accumulate(

Table.ToRecords( ReplacementTable ),

[Consumer_ID],

( valueToReplace, replaceOldNewRecord ) =>

Text.Replace( valueToReplace,

replaceOldNewRecord[Old],

replaceOldNewRecord[New]

)

)


Подробное рассмотрение функции List.Accumulate выходит за рамки этой статьи . Для этого вы можете найти отдельные статьи. Просто обратите внимание, как:

  • Строка 2 ссылается на имя запроса таблицы перевода, которое является replaceTable .
  • В строке 3 содержится имя столбца с заменяемыми текстовыми значениями, то есть [Consumer_ID].
  • Строки 6 и 7 ссылаются на столбцы таблицы замены, а именно [ Old ] и [ New ].
  • Значения valueToReplace и replaceOldNewRecord не являются реальными именами таблиц или столбцов, а являются временными переменными внутри функции, созданной в строке 4.

На данный момент вы можете добавить это в пользовательский столбец, чтобы увидеть, как оно заменяет соответствующие значения.

Вы можете повысить производительность запросов, обернув replaceTable в функцию List.Buffer . Это предотвращает многократный доступ Power Query к внешним запросам, сокращая время загрузки.

= List.Accumulate(

List.Buffer( Table.ToRecords( ReplacementTable ) ),

[Consumer_ID],

( valueToReplace, replaceOldNewRecord ) =>

Text.Replace( valueToReplace,

replaceOldNewRecord[Old],

replaceOldNewRecord[New]

)

)


И последнее, но не менее важное: вы можете массово заменить несколько значений за один шаг, заменив значения с помощью:

= Table.ReplaceValue(

#"Changed Type",

each [Consumer_ID],

each List.Accumulate(

List.Buffer( Table.ToRecords( ReplacementTable ) ),

[Consumer_ID],

( valueToReplace, replaceOldNewRecord ) =>

Text.Replace( valueToReplace,

replaceOldNewRecord[Old],

replaceOldNewRecord[New] ) ),

Replacer.ReplaceText,

{"Consumer_ID"}

)


А пока не беспокойтесь слишком сильно о тонкостях используемой здесь функции. Это заслуживает большего внимания в отдельной статье.

Заменить значения: без учета регистра

По умолчанию Power Query заменяет значения операцией с учетом регистра. В этой главе вы узнаете, как игнорировать регистр при замене значений. Замена текста rIck на Rick в приведенном ниже наборе данных заменяет только второе значение. И пользовательский интерфейс не дает вам никаких возможностей для замены без учета регистра. Итак, какие методы вы можете использовать?

Содержащее ключевое слово

Чтобы проверить, содержит ли ваша строка значение для поиска , а затем заменить все содержимое ячейки, вы можете составить формулу условной замены в Power Query с помощью:

= Table.ReplaceValue(

#"Changed Type",

each [Consumer_ID],

each if Text.Contains([Consumer_ID], "rick", Comparer.OrdinalIgnoreCase )

then "Rick"

else [Consumer_ID],

Replacer.ReplaceValue,

{"Consumer_ID"}

)


Формула проверяет, содержит ли столбец текст и не учитывает ли он регистр. Часть Comparer.OrdinalIgnoreCase указывает функции Text.Contains игнорировать регистр. Это означает, что различия в заглавных буквах не влияют на ваш результат. При наличии совпадения значение меняется на Rick , в противном случае остается исходное значение.

А что, если вы хотите заменить текст без учета регистра не только для Рика, но и для нескольких ключевых слов?

Массовая замена с использованием ручного списка

Возможно, вам захочется провести большую очистку набора данных, если он полон орфографических ошибок. И даже при множественных заменах может помочь Power Query. Чтобы заменить несколько элементов без учета регистра, вы можете использовать функцию List.ReplaceMatchingItems .

Функция имеет два обязательных аргумента и необязательный:

= List.ReplaceMatchingItems(

list as list, // list to perform replacements on

replacements as list, // list of replacements

optional equationCriteria as any) // determines case-sensitivity

as list


Давайте сначала рассмотрим замену текстового значения, состоящего из одного слова. Для начала вам нужно заменить Rick на RICk . Давайте сначала попробуем это в пользовательском столбце, добавив:

= List.ReplaceMatchingItems(

{ [Consumer_ID] },

{ { "Rick", "RICk" } }

)


На человеческом языке это предписывает заменить значения в столбце [Consumer_ID], равные Rick ( с учетом регистра ), на RICk .

Результатом операции замены является список. Чтобы проверить один из этих элементов списка, вы можете щелкнуть пробел внутри ячейки, содержащей список (не нажимайте на само слово «список»). После этого внизу появится предварительный просмотр. Этот предварительный просмотр показывает, что ваша операция замены прошла успешно.

Чтобы упростить проверку других значений, вы можете обернуть формулу в Text.Combine . Эта функция принимает список в качестве аргумента и объединяет все значения в списке. Поскольку у нас есть одно значение, оно просто вернет это значение.

= Text.Combine(

List.ReplaceMatchingItems(

{ [Consumer_ID] },

{ { "Rick", "RICk" } }

)

)


Как показано на рисунке выше, код заменяет только значение в строке 1. Это происходит потому, что по умолчанию функция List.ReplaceMatchingItems заменяет значения с учетом регистра.

Чтобы исправить это, вы можете добавить третий необязательный аргумент , который является критерием уравнения или функцией сравнения. Функция сравнения Comparer.OrdinalIgnoreCase указывает функциям игнорировать заглавные буквы при замене значений. Вы обнаружите, что эта функция сравнения присутствует во многих других функциях списков в Power Query .

Ваша скорректированная формула выглядит следующим образом:

= Text.Combine(

List.ReplaceMatchingItems(

{ [Consumer_ID] },

{ { "Rick", "RICk" } },

Comparer.OrdinalIgnoreCase

)

)


В результате получается преобразованный набор данных, как показано ниже:

Это заменяет ячейки, все содержимое которых равно слову Rick , игнорируя регистр. Теперь у вас есть только еще одно требование. Вы хотите иметь возможность выполнять несколько замен в одной формуле. Этот последний шаг вы можете сделать, расширив списки значениями для замены.

List.ReplaceMatchingItems принимает « список из двух значений: старого значения и нового значения, представленных в списке », как показано в документации. В примере это означает, что мы можем добавить больше элементов во внешний список второго аргумента:

= Text.Combine(

List.ReplaceMatchingItems(

{ [Consumer_ID] },

{ { "Rick", "RICk" },

{ "GorillaBI", "GORILLABi" },

{ "RickdeGroot", "RICKDEGROOt" } },

Comparer.OrdinalIgnoreCase

)

)


И при этом вы теперь заменили несколько элементов без учета регистра на совпадение всего содержимого ячейки.

Массовая замена с использованием таблицы перевода

Возможно, вы заметили, что этот список по-прежнему необходимо поддерживать вручную. Что не очень удобно при больших объемах замен, или при обслуживании. Итак, как мы можем это улучшить? Один из способов — использовать таблицу замены для сохранения заменяемых значений. Допустим, у вас есть таблица replaceTable со столбцами Old и New .

Вместо ручного списка замен это можно автоматизировать. Для этого вам нужно узнать немного больше о функции List.Zip . Функция принимает список списков в качестве аргумента, а затем преобразует его в другой список. Простой пример этого будет ниже:

= List.Zip( { {"a", "b", "c", "d"}, {1, 2, 3, 4} } )


Результат функции:

{ { "a", 1 },

{ "b", 2 },

{ "c", 3 },

{ "d", 4 } }


Он берет первый элемент первого списка и сопоставляет его с первым элементом второго списка. А затем он делает это для второго значения каждого списка и третьего, создавая список пар. И эти пары представляют собой инструкции, необходимые для замены значений. Наш исходный код выглядел так:

= Text.Combine(

List.ReplaceMatchingItems(

{ [Consumer_ID] },

{ { "Rick", "RICk" },

{ "GorillaBI", "GORILLABi" },

{ "RickdeGroot", "RICKDEGROOt" } },

Comparer.OrdinalIgnoreCase

)

)


Строки 4-6 представляют аналогичный список пар . Имея это в виду, вы можете использовать List.Zip для создания списка с заменами. Для этого вы ссылаетесь на таблицу замены и добавляете за ней имя столбца. Ссылка на столбец всегда приводит к появлению списка значений этого столбца. И этот список мы можем предоставить List.Zip .

= List.Zip( { ReplacementTable[Old],

ReplacementTable[New] } )

// With a 2 column replacement table you can achieve the same with:

= Table.ToColumns( ReplacementTable )


Это создает окончательную формулу:

= Text.Combine(

List.ReplaceMatchingItems(

{ [Consumer_ID] },

List.Zip( { ReplacementTable[Old],

ReplacementTable[New] } ),

Comparer.OrdinalIgnoreCase

)

)


По соображениям производительности вы можете затем обернуть List.Zip в функцию List.Buffer :

= Text.Combine(

List.ReplaceMatchingItems(

{ [Consumer_ID] },

List.Buffer(

List.Zip( { ReplacementTable[Old],

ReplacementTable[New] } )

),

Comparer.OrdinalIgnoreCase

)

)


Массовая замена предложений

До сих пор замены без учета регистра выполнялись для текстовых значений, состоящих из одного слова. Если ваш текст состоит из предложения, состоящего из нескольких слов, вам придется немного скорректировать подход. Поскольку вы хотите заменить значение без учета регистра, важно, чтобы замена происходила по всему слову, независимо от того, какая буква написана с заглавной буквы. Этого можно добиться, сначала разделив предложение на слова, затем выполнив регистронезависимую замену всего значения и снова соединив их вместе.

= Text.Combine(

List.ReplaceMatchingItems (

Text.Split( [Text], " " ) ,

List.Buffer( List.Zip( Table.ToColumns( ReplacementTable ) ) ),

Comparer.OrdinalIgnoreCase

),

" ")


В качестве альтернативы вы можете использовать следующее решение, предложенное Биллом Шишем, которое немного медленнее.

= let ReplaceTbl = Table.Buffer( ReplacementTable ) in

Text.Combine(

List.Transform(

Text.Split( [Text], " " ),

each try ReplaceTbl[New]{ List.PositionOf( ReplaceTbl[Old], _) }

otherwise _ ),

" " )


Массовая замена подстрок

И последнее, но не менее важное: если вы ищете способ легко заменить несколько текстовых значений в Power Query, у вас есть хорошие новости! List.Accumulate здесь, чтобы спасти положение. Если вы новичок в этой функции, это руководство поможет вам начать работу .

List.Accumulate позволяет многократно выполнять такие действия, как массовая замена подстрок в наборе данных.

Представьте себе: у вас есть набор данных со столбцом под названием «Значение» и множество замен, которые нужно сделать. Вот изящный способ сделать это:

List.Accumulate (

{ { "A", "Ab" }, { "B", "Bc" }, { "C", "Cd" }, { "D", "De" } },

Source,

( state, current ) =>

Table.ReplaceValue (

state,

current{0},

current{1},

Replacer.ReplaceText, { "Value" } )

)[MyTable]


Итак, что здесь происходит? List.Accumulate перебирает комбинации значений, которые необходимо заменить. Все начинается с замены «А» на «Аб», затем «Б» на «Вс» и так далее. Просто обязательно настройте начальные начальные значения, иначе Power Query может их пропустить.

Аналогичным образом вы можете использовать таблицу перевода, изменив код на:

List.Accumulate (

List.Zip( { replacementTable[Old], replacementTable[New] } ),

Source,

( state, current ) =>

Table.ReplaceValue (

state,

current{0},

current{1},

Replacer.ReplaceText, { "Value" } )

)[MyTable]


Вот и все! Теперь вы можете заменять подстроки в Power Query массово, не беспокоясь. Удачи!
