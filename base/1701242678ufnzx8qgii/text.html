Как вернуться к последнему коммиту после проверки предыдущего коммита?

Если вы знаете, что коммит, к которому вы хотите вернуться, является главой какой-либо ветки или помечен тегом, вы можете просто

git checkout branchname

Вы также можете использовать его git reflog, чтобы увидеть, на какие другие коммиты ваш HEAD (или любой другой ref) указывал в прошлом.

В более новых версиях Git, если вы запускали git checkoutили что-то еще, чтобы переместить HEADодин раз, вы также можете сделать

git checkout -

чтобы вернуться туда, где он был до последней проверки. Это было мотивировано аналогией с идиомой оболочки, cd -требующей возврата в любой рабочий каталог, в котором вы находились ранее.

Для получения информации в более удобном виде, можно воспользоваться командой git log -g, которая выведет лог записей из reflog в привычном формате:

git log -g
