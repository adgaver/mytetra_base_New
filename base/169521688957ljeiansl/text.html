Возвращает экземпляр постоянного контекста браузера.

Запускает браузер, который использует постоянное хранилище, расположенное по адресу, userDataDirи возвращает единственный контекст. Закрытие этого контекста автоматически закроет браузер.

Применение

await browserType.launchPersistentContext(userDataDir);
await browserType.launchPersistentContext(userDataDir, options);


Аргументы

  • userDataDir нить#

    Путь к каталогу пользовательских данных, в котором хранятся данные сеанса браузера, такие как файлы cookie и локальное хранилище. Более подробная информация о Chromium и Firefox . Обратите внимание, что каталог пользовательских данных Chromium является родительским каталогом «Пути к профилю», видимого по адресу chrome://version. Передайте пустую строку, чтобы вместо этого использовать временный каталог.

  • options Объект (необязательно)

    • acceptDownloads логическое значение (необязательно)#

      Следует ли автоматически загружать все вложения. По умолчанию trueпринимается все загрузки.

    • args Массив < строка > (необязательно)#

      Дополнительные аргументы для передачи экземпляру браузера. Список флагов Chromium можно найти здесь .

    • baseURL строка (необязательно)#

      При использовании page.goto() , page.route() , page.waitForURL() , page.waitForRequest() или page.waitForResponse() он принимает во внимание базовый URL-адрес, используя URL()конструктор для создания соответствующего URL-адреса. Не установлено по умолчанию. Примеры:

      • baseURL: http://localhost:3000и переход к /bar.htmlрезультатам вhttp://localhost:3000/bar.html
      • baseURL: http://localhost:3000/foo/и переход к ./bar.htmlрезультатам вhttp://localhost:3000/foo/bar.html
      • baseURL: http://localhost:3000/foo(без косой черты) и переход к ./bar.htmlрезультатамhttp://localhost:3000/bar.html

    • bypassCSP логическое значение (необязательно)#

      Переключает обход Content-Security-Policy страницы. По умолчанию false.

    • channel строка (необязательно)#

      Канал распространения браузера. Поддерживаемые значения: «chrome», «chrome-beta», «chrome-dev», «chrome-canary», «msedge», «msedge-beta», «msedge-dev», «msedge-canary». Узнайте больше об использовании Google Chrome и Microsoft Edge .

    • chromiumSandbox логическое значение (необязательно)#

      Включите песочницу Chromium. По умолчанию false.

    • colorScheme null |"светлый"|"темный"|"без предпочтений" (необязательно)#

      Эмулирует 'prefers-colors-scheme'функцию мультимедиа. Поддерживаемые значения: 'light', 'dark', 'no-preference'. Дополнительную информацию см. в разделе page.emulateMedia() . Передача nullсбрасывает эмуляцию к системным настройкам по умолчанию. По умолчанию 'light'.

    • deviceScaleFactor номер (необязательно)#

      Укажите масштабный коэффициент устройства (можно рассматривать как dpr). По умолчанию 1. Узнайте больше об эмуляции устройств с помощью масштабного коэффициента устройства .

    • devtools логическое значение (необязательно)#

      Только для Chromium. Следует ли автоматически открывать панель инструментов разработчика для каждой вкладки. Если эта опция есть true, она headlessбудет установлена false.

    • downloadsPath строка (необязательно)#

      Если указано, принятые загрузки загружаются в этот каталог. В противном случае создается временный каталог, который удаляется при закрытии браузера. В любом случае загрузки удаляются при закрытии контекста браузера, в котором они были созданы.

    • env Объект < строка , строка | номер | логическое значение > (необязательно)#

      Укажите переменные среды, которые будут видны браузеру. По умолчанию process.env.

    • executablePath строка (необязательно)#

      Путь к исполняемому файлу браузера, который будет запускаться вместо встроенного. Если executablePathэто относительный путь, то он разрешается относительно текущего рабочего каталога. Обратите внимание, что Playwright работает только с включенными в комплект Chromium, Firefox или WebKit, используйте на свой страх и риск.

    • extraHTTPHeaders Объект < строка , строка > (необязательно)#

      Объект, содержащий дополнительные заголовки HTTP, которые будут отправляться с каждым запросом. По умолчанию нет.

    • forcedColors ноль | «активный» | «нет» (необязательно)#

      Эмулирует 'forced-colors'функцию мультимедиа. Поддерживаемые значения: 'active', 'none'. Дополнительную информацию см. в разделе page.emulateMedia() . Передача nullсбрасывает эмуляцию к системным настройкам по умолчанию. По умолчанию 'none'.

    • geolocation Объект (необязательно)#

      • latitude число

        Широта от -90 до 90.

      • longitude число

        Долгота между -180 и 180.

    • handleSIGTERM логическое значение (необязательно)#

    • hasTouch логическое значение (необязательно)#

      Указывает, поддерживает ли область просмотра события касания. По умолчанию ложь. Узнайте больше об эмуляции мобильных устройств .

    • headless логическое значение (необязательно)#

      Запускать ли браузер в безгласном режиме. Более подробная информация о Chromium и Firefox . По умолчанию, trueесли не devtoolsуказана опция true.

    • httpCredentials Объект (необязательно)#

      • username нить

      • password нить

      • origin строка (необязательно)

        Ограничить отправку учетных данных http в определенном источнике (схема://хост:порт).

      Учетные данные для HTTP-аутентификации . Если источник не указан, имя пользователя и пароль отправляются на любые серверы при несанкционированных ответах.

    • ignoreDefaultArgs логическое | Массив < строка > (необязательно)#

      Если true, Playwright не передает аргументы собственной конфигурации и использует только те, которые находятся в args. Если указан массив, отфильтровываются заданные аргументы по умолчанию. Опасный вариант; используйте с осторожностью. По умолчанию false.

    • ignoreHTTPSErrors логическое значение (необязательно)#

      Игнорировать ли ошибки HTTPS при отправке сетевых запросов. По умолчанию false.

    • isMobile логическое значение (необязательно)#

      meta viewportУчитывается ли тег и включены ли сенсорные события. isMobile является частью устройства, поэтому вам не нужно настраивать его вручную. По умолчанию используется falseи не поддерживается в Firefox. Узнайте больше об эмуляции мобильных устройств .

    • javaScriptEnabled логическое значение (необязательно)#

      Включить или нет JavaScript в контексте. По умолчанию true. Узнайте больше об отключении JavaScript .

Локаль повлияет на navigator.languageзначение, Accept-Languageзначение заголовка запроса, а также на правила форматирования чисел и дат. По умолчанию используется локаль системы по умолчанию. Узнайте больше об эмуляции в нашем руководстве по эмуляции .

    • logger Регистратор (необязательно)#

      Эмулировать ли работу сети в автономном режиме. По умолчанию false. Узнайте больше об эмуляции сети .

    • permissions Массив < строка > (необязательно)#

      Список разрешений, которые можно предоставить всем страницам в этом контексте. Дополнительную информацию см. в разделе BrowserContext.grantPermissions() . По умолчанию нет.

    • proxy Объект (необязательно)#

      • server нить

        Прокси, который будет использоваться для всех запросов. Поддерживаются прокси-серверы HTTP и SOCKS, например http://myproxy.com:3128или socks5://myproxy.com:3128. Краткая форма myproxy.com:3128считается HTTP-прокси.

      • bypass строка (необязательно)

        Необязательные домены, разделенные запятыми, для обхода прокси, например ".com, chromium.org, .domain.com".

      • username строка (необязательно)

        Необязательное имя пользователя, которое можно использовать, если HTTP-прокси требует аутентификации.

      • password строка (необязательно)

        Необязательный пароль для использования, если HTTP-прокси требует аутентификации.

      Настройки сетевого прокси.

    • recordHar Объект (необязательно)#

      • omitContent логическое значение (необязательно)

        Дополнительный параметр, позволяющий контролировать, следует ли пропускать содержимое запроса из HAR. По умолчанию false. Устарело, contentвместо этого используйте политику.

      • content"опустить"|"вставить"|"прикрепить" (необязательно)

        Дополнительная настройка для управления управлением содержимым ресурсов. Если omitуказано, содержимое не сохраняется. Если attachуказано, ресурсы сохраняются в виде отдельных файлов или записей в ZIP-архиве. Если embedуказано, содержимое сохраняется внутри файла HAR в соответствии со спецификацией HAR. По умолчанию attachдля .zipвыходных файлов и embedдля всех других расширений файлов.

      • path нить

        Путь в файловой системе для записи файла HAR. Если имя файла заканчивается на .zip, content: 'attach'используется по умолчанию.

      • mode"полный"|"минимальный" (необязательно)

        Если установлено значение minimal, запишите только информацию, необходимую для маршрутизации из HAR. При этом не учитываются размеры, время, страница, файлы cookie, безопасность и другие типы информации HAR, которые не используются при воспроизведении из HAR. По умолчанию full.

      • urlFilter строка | RegExp (необязательно)

        Шаблон glob или регулярного выражения для фильтрации запросов, хранящихся в HAR. Если baseURLбыли предоставлены параметры контекста и переданный URL-адрес является путем, он объединяется через new URL()конструктор. По умолчанию нет.

      Включает запись HARrecordHar.path для всех страниц в файл. Если не указано, HAR не записывается. Обязательно дождитесь BrowseContext.close() для сохранения HAR.

    • recordVideo Объект (необязательно)#

      • dir нить

        Путь к каталогу, в который будут помещены видео.

      • size Объект (необязательно)

        • width число

          Ширина видеокадра.

        • height число

        Дополнительные размеры записанных видео. Если не указано, размер будет равен viewportуменьшенному до 800x800. Если viewportэто не настроено явно, размер видео по умолчанию равен 800x450. Фактическое изображение каждой страницы будет при необходимости уменьшено до указанного размера.

      Включает запись видео для всех страниц recordVideo.dirкаталога. Если не указано иное, видео не записываются. Обязательно дождитесь BrowseContext.close() для сохранения видео.

    • reducedMotion null |"сокращение"|"без предпочтений" (необязательно)#

      Эмулирует 'prefers-reduced-motion'функцию мультимедиа. Поддерживаемые значения: 'reduce', 'no-preference'. Дополнительную информацию см. в разделе page.emulateMedia() . Передача nullсбрасывает эмуляцию к системным настройкам по умолчанию. По умолчанию 'no-preference'.

    • screen Объект (необязательно)#

      • width число

        ширина страницы в пикселях.

      • height число

        высота страницы в пикселях.

      Эмулирует постоянный размер экрана окна, доступный на веб-странице через window.screen. Используется только тогда, когда viewportустановлен .

    • serviceWorkers"разрешить"|"заблокировать" (необязательно)#

      Разрешить ли сайтам регистрировать Service Workers. По умолчанию 'allow'.

      • 'allow': Сервис-воркеры могут быть зарегистрированы.
      • 'block': Драматург заблокирует всю регистрацию сервисных работников.

    • slowMo номер (необязательно)#

      Замедляет работу драматурга на указанное количество миллисекунд. Полезно, чтобы вы могли видеть, что происходит.

    • strictSelectors логическое значение (необязательно)#

      Если установлено значение true, включает режим строгого селектора для этого контекста. В режиме строгих селекторов все операции с селекторами, которые подразумевают один целевой элемент DOM, будут вызываться, когда селектору соответствует более одного элемента. Эта опция не влияет ни на какие API-интерфейсы локаторов (локаторы всегда строгие). По умолчанию false. См. раздел «Локатор» , чтобы узнать больше о строгом режиме.

    • timeout номер (необязательно)#

      Максимальное время в миллисекундах ожидания запуска экземпляра браузера. По умолчанию 30000(30 секунд). Нажмите 0, чтобы отключить тайм-аут.

    • timezoneId строка (необязательно)#

      Изменяет часовой пояс контекста. Список поддерживаемых идентификаторов часовых поясов см. в файле MetaZones.txt ICU . По умолчанию используется системный часовой пояс.

    • tracesDir строка (необязательно)#

      Если указано, трассировки сохраняются в этот каталог.

    • userAgent строка (необязательно)#

      Конкретный пользовательский агент, который будет использоваться в этом контексте.

      • height число

    • videosPath строка (необязательно)#

      УСТАРЕЛО

      Используйте recordVideoвместо этого.

    • viewport ноль | Объект (необязательно)#

      • width число

        ширина страницы в пикселях.

      • height число

        высота страницы в пикселях.

      Эмулирует единообразную область просмотра для каждой страницы. По умолчанию используется область просмотра 1280x720. Используйте null, чтобы отключить последовательную эмуляцию области просмотра. Узнайте больше об эмуляции области просмотра .

      ПРИМЕЧАНИЕ

      Значение nullисключается из предустановок по умолчанию, делает область просмотра зависимой от размера окна хоста, определенного операционной системой. Это делает выполнение тестов недетерминированным.
