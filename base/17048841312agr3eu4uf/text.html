Введение

Освоение текстовых функций в Power Query необходимо всем, кто работает с текстовыми значениями. Это может сделать процесс очистки данных проще и эффективнее. С помощью правильных текстовых функций вы можете быстро и легко преобразовать текстовые данные в нужный формат.

Понимание текстовых функций

Давайте углубимся в то, как работает каждая функция. В следующих главах мы рассмотрим все: от вставки текста, удаления текста, извлечения текста, преобразования текста и многого другого.

Вставить текст

В Power Query вы можете вставлять текст с различными функциями. Функция Text.Insert позволяет добавлять текстовое значение в существующее текстовое значение в указанной позиции . Эта позиция начинается с индекса 0.

= Text.Insert( "Fish", 0, "Big " ) // Returns "Big Fish"

= Text.Insert( "Arc", 3, "h" ) // Returns "Arch"

= Text.Insert( "Ab", Text.Length( "Ab"), "c" ) // Combines functions, returns "Abc"


Бывают случаи, когда вам нужно, чтобы текстовое значение имело определенную длину. Чтобы решить эту проблему, вы можете вставить значение в начало или конец строки. Это могут сделать функции Text.Padstart и Text.PadEnd .

Обе текстовые функции имеют три аргумента. Сначала текстовое значение , затем желаемое количество символов для новой строки и необязательный символ, используемый для заполнения .

Символом заполнения по умолчанию является пробел. Он используется, когда третий аргумент оставлен пустым. Тем не менее, вы можете предоставить третьему аргументу желаемый символ заполнения. Типичное использование — добавление ведущих нулей к значению.

= Text.PadStart( "Gorilla", 8 ) // Returns " Gorilla"

= Text.PadStart( "5440", 7, "0" ) // Returns "0005440"

= Text.PadStart( "544", 7, "0" ) // Returns "0000544"

= Text.PadStart( "54", 7, "0" ) // Returns "0000054"

= Text.PadEnd( "Gorilla", 8 ) // Returns "Gorilla "

= Text.PadEnd( "5440", 7, "0" ) // Returns "5440000"

= Text.PadEnd( "544", 7, "0" ) // Returns "5440000"

= Text.PadEnd( "54", 7, "0" ) // Returns "5400000"


Удалить текст

Функция Text.Remove позволяет удалить все вхождения символа. Вы можете удалить буквы, цифры или любой другой символ. Функция имеет два аргумента.

Первый аргумент принимает текстовое значение , второй аргумент требует ввода одного или нескольких символов, чтобы удалить ввод в виде односимвольных строк . Вы можете ввести это как одно текстовое значение или как список, разделенный запятыми, с односимвольными строками.

// You can remove a single character or multiple.

= Text.Remove( "Rick de Groot", "o" ) // returns "Rick de grt"

= Text.Remove( "bigorilla", { "i", "e", "o" } ) // returns "bgrlla"

// The Text.Remove function is capital sensitive.

= Text.Remove( "Rick", "R" ) // Returns "ick"

= Text.Remove( "Rick", "r" ) // Returns "Rick"

// And accepts numbers, letters but also special characters.

= Text.Remove( "A/B_C", { "/", "_" } ) // Returns "ABC"

// You can also use a shortcut to generate the values to remove.

= Text.Remove( "power-BI99", { "A".."Z" } ) // "power-99"

= Text.Remove( "power-BI99", { "a".."z" } ) // "-BI99"

= Text.Remove( "power-BI99", { "a".."z", "A".."Z" } ) // "-99"

= Text.Remove( "power-BI99", { "0".."9" } ) // "power-BI"


В предыдущих примерах из строки удаляются все предоставленные символы. Могут быть случаи, когда вы захотите удалить предоставленные символы в начале или конце строки. В этих случаях вы можете использовать Text.TrimStart и Text.TrimEnd .

= Text.TrimStart( "***gorilla***", "*" ) // Returns "gorilla***"

= Text.TrimStart( "* * gorilla", {" ", "*"} ) // Returns "gorilla"

= Text.TrimEnd( "***gorilla***", "*" ) // Returns "***gorilla"

= Text.TrimEnd( "!845pqr", {"a".."z"} ) // Returns "!845"


Хотя функция Text.RemoveRange очень похожа на Text.Remove, она делает нечто совершенно иное. Функция позволяет указать смещение и количество символов для удаления . Затем он удаляет это количество символов, начиная со смещения.

= Text.RemoveRange( "9-AR-C-85", 0, 5 ) // Returns "C-85"

= Text.RemoveRange( "9-AR-C-85", 3, 3 ) // Returns "9-A-85"


Извлечь текст

Следующая категория текстовых функций ориентирована на извлечение значений из строк.

Одна из моих любимых функций — Text.Select . Вы можете указать функции сохранять все вхождения одного или нескольких заданных символов. И при этом это полная противоположность функции Text.Remove .

Функция Text.Select имеет два аргумента. Сначала ему нужно текстовое значение , а вторые аргументы содержат сохраняемые символы .

= Text.Select( "jUxL2jE98Z", { "0" .. "9" } ) // Returns "298"

= Text.Select( "jUxL2jE98Z", { "a" .. "z" } ) // Returns "jxj"

= Text.Select( "jUxL2jE98Z", { "A" .. "Z" } ) // Returns "ULEZ"

= Text.Select( "jUxL2jE98Z", { "x", "E", "9" } ) // Returns "xE9"


Функция Text.Start извлекает первые символы строки, тогда как функция Text.End извлекает последние символы текстового значения. Обе функции принимают текстовое значение в качестве первого аргумента и требуют количество извлекаемых символов в качестве второго аргумента.

= Text.Start( "Life is good", 4 ) // Returns "Life"

= Text.Start( "Life is good", 7 ) // Returns "Life is"

= Text.End( "Life is good", 4 ) // Returns "good"

= Text.End( "Life is good", 7 ) // Returns "is good"


Иногда вам может потребоваться вернуть значения в определенной позиции строки. Вы можете вернуть один символ, используя функцию Text.At. Для этого требуется текстовое значение в качестве первого аргумента и позиция смещения желаемого символа в качестве второго аргумента.

Чтобы вернуть несколько значений, вы можете использовать функцию Text.Range в Power Query. Он принимает текстовое значение в качестве первого аргумента и позицию смещения в качестве второго. При желании вы можете указать третий аргумент, чтобы указать количество возвращаемых символов . Если этот аргумент опущен, функция возвращает все символы, начиная со смещения.

// Text.At always returns a single character

= Text.At( "Run@", 3 ) // Returns "@"

= Text.At( " - - - ", 1 ) // Returns "-"

// Text.Range can return a single character

= Text.Range( "Run@", 3 ) // Returns "@"

= Text.Range( "Run@", 3, 1 ) // Returns "@"

= Text.Range( "Run@", 5, 1 ) // Returns error: offset is out of range

// Or provides you with a range of values.

= Text.Range( " - - - ", 1 ) // Returns "- - - "

= Text.Range( " - - - ", 1, 3 ) // Returns "- -"

= Text.Range( "I am Groot", 2 ) // Returns "am Groot"

= Text.Range( "I am Groot", 2, 2 ) // Returns "am"


Функция Text.Middle работает идентично функции Text.Range. За одним исключением. Если функция Text.Middle выходит за пределы диапазона, она все равно возвращает значение. Однако функция Text.Range выдает ошибку: « Expression.Error: Аргумент count выходит за пределы допустимого диапазона. «

= Text.Middle( "Hello World", 6, 10 ) // Returns World

= Text.Range( "Hello World", 6, 10 ) // Returns Error


До сих пор функциям требовалось определенное смещение или количество символов для извлечения. Другой набор функций извлекает значения на основе разделителей .

Функция Text.AfterDelimiter извлекает весь текст после указанного разделителя, тогда как Text.BeforeDelimiter извлекает все, что находится до разделителя. Обе функции имеют два обязательных аргумента и один необязательный аргумент.

Первый аргумент требует текстового значения , а второй аргумент использует разделитель для поиска . По умолчанию обе функции будут искать первый найденный разделитель.

Если вы не хотите искать первый разделитель, вы можете использовать третий необязательный аргумент, чтобы указать числовые разделители, которые следует пропустить перед возвратом значения. Вы даже можете указать, с какой стороны пропускать разделители, используя RelativePosition.FromStart и RelativePosition.FromEnd .

= Text.BeforeDelimiter( "x-87-v", "-" ) // Returns "x"

= Text.BeforeDelimiter( "x-87-v", "-", 0 ) // Returns "x"

= Text.BeforeDelimiter( "x-87-v", "-", 1 ) // Returns "x-87"

// You can indicate from which side to search for the delimiter

// by using RelativePosition.FromStart and RelativePosition.FromEnd

// The functions by default use RelativePosition.FromStart

// Below formulas return: 1) "x", 2) "x", 3 ) "x-87", 4) "x"

1 = Text.BeforeDelimiter( "x-87-v", "-", 0 )

2 = Text.BeforeDelimiter( "x-87-v", "-", { 0, RelativePosition.FromStart } )

3 = Text.BeforeDelimiter( "x-87-v", "-", { 0, RelativePosition.FromEnd } )

4 = Text.BeforeDelimiter( "x-87-v", "-", { 1, RelativePosition.FromEnd } )

// Text.AfterDelimiter works with similar arguments.

= Text.AfterDelimiter( "x-87-v", "-" ) // Returns "87-v"

= Text.AfterDelimiter( "x-87-v", "-", 0 ) // Returns "87-v"

= Text.AfterDelimiter( "x-87-v", "-", 1 ) // Returns "v"

// Below formulas return: 5) "87-v", 6) "87-v", 7) "v", 8) "87-v"

5 = Text.AfterDelimiter( "x-87-v", "-", 0 )

6 = Text.AfterDelimiter( "x-87-v", "-", { 0, RelativePosition.FromStart } )

7 = Text.AfterDelimiter( "x-87-v", "-", { 0, RelativePosition.FromEnd } )

8 = Text.AfterDelimiter( "x-87-v", "-", { 1, RelativePosition.FromEnd } )


В показанных примерах рассматривается текст до или после разделителя. Однако одна текстовая функция все еще отсутствует. Функция Text.BetweenDelimiters извлекает текст между указанным начальным и конечным разделителем .

Как и две ее сестры, эта функция принимает начальный и конечный индекс , указывающий количество значений, которые нужно пропустить . И вы можете превратить оба значения в индекс списка, чтобы также указать, следует ли вам пропускать значения в конце или начале ввода.

= Text.BetweenDelimiters( "x-8-v-0", "-", "-" ) // Returns "8"

= Text.BetweenDelimiters( "x-8-v-0", "-", "v" ) // Returns "8-"

= Text.BetweenDelimiters( "x-8-v-0", "-", "0" ) // Returns "8-v-"

= Text.BetweenDelimiters( "x-8-v-0", "-", "-", 1 ) //Returns "v"

= Text.BetweenDelimiters( "x-8-v-0", "-", "-", 2 ) // Returns "0"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-", 1, 0 ) // Returns "v"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-", 1, 1 ) // Returns "v-0"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-", 0, 1 ) // Returns "8-v"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-", 0, 2 ) // Returns "8-v-0"

// Below three expression are identical. The function uses

// RelativePosition.FromStart and an index of 0 by default.

= Text.BetweenDelimiters( "x-8-v-0", "-", "-" ) // Returns "8"

= Text.BetweenDelimiters( "x-8-v-0", "-", "-", 0, 0 ) // Returns "8"

= Text.BetweenDelimiters( "x-8-v-0", "-", "-",

{ 0, RelativePosition.FromStart },

{ 0, RelativePosition.FromStart } ) // Returns "8"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-",

{ 1, RelativePosition.FromStart },

{ 0, RelativePosition.FromStart } ) // Returns "v"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-",

{ 2, RelativePosition.FromStart },

{ 0, RelativePosition.FromStart } ) // Returns "0"

// Once you reach the end of the input increasing the index has no impact

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-",

{ 2, RelativePosition.FromStart },

{ 1, RelativePosition.FromStart }) // Returns "0"

// The function looks for the second "-", and then from there looks back

// to the start of the input value

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-",

{ 1, RelativePosition.FromStart },

{ 2, RelativePosition.FromEnd } ) // Returns "x-8"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-",

{ 2, RelativePosition.FromStart },

{ 2, RelativePosition.FromEnd } ) // Returns "x-8-v"

= Text.BetweenDelimiters( "x-8-v-0", "-" ,"-",

{ 2, RelativePosition.FromEnd },

{ 2, RelativePosition.FromEnd } ) // Returns "x"


Преобразование текста

В этой главе основное внимание уделяется текстовым функциям, которые помогают преобразовывать и очищать текстовые значения. Некоторые из них относительно просты с одним или двумя аргументами. Давайте посмотрим, как вы можете использовать Text.Lower , Text.Upper , Text.Proper , Text.Trim , Text.Clean , Text.Reverse , Text.Repeat и Text.Combine .

= Text.Lower( "abcDEF" ) // Returns "abcdef"

= Text.Upper( "abcDEF" ) // Returns "ABCDEF"

// Capitalizes the first letter of each word

= Text.Proper( "abcDEF" ) // Returns "Abcdef"

= Text.Proper( "i like cats and dogs" ) // Returns "I Like Cats And Dogs"

// Removes control characters from a string

= Text.Clean( "R#(lf)i#(cr)ck" ) // Returns "Rick"


Вы также можете сосредоточиться на удалении пробелов, переворачивании текста или их объединении.

// Removes leading & trailing characters, white space by default

= Text.Trim( " gorilla " ) // Returns "gorilla"

= Text.Trim( " gorilla ", " " ) // Returns "gorilla"

= Text.Trim( "- -gorilla- -", "-" ) // Returns " -gorilla- "

= Text.Trim( "- -gorilla- -", "-" ) // Returns " -gorilla- "

= Text.Trim( "- -gorilla- - ", { "-", " " } ) // Returns "gorilla"

= Text.Reverse( "gorilla" ) // Returns "allirog"

= Text.Repeat( "abc", 3 ) // Returns "abcabcabc"

// The second argument allows you to input an optional separator

= Text.Combine( { "abc", "123" } ) // Returns "abc123"

= Text.Combine( { "abc", "123" }, "-" ) // Returns "abc-123"

= Text.Combine( { "ab", "12", "." }, "-" ) // Returns "ab-12-."


Уникальная текстовая функция для объединения и форматирования ваших текстов — Text.Format . В качестве аргументов функция сначала принимает строку формата , затем аргумент из списка или записи и, при необходимости, код культуры. Аналогичных результатов можно добиться с помощью функции Text.Combine . Однако преимуществом этой функции является то, что вы можете предоставить код культуры.

= Text.Format( "The #[Subject] wants a #[Food].",

[ Subject = "Gorilla", Food = "Banana" ] )

// Returns values from a record: "The Gorilla wants a Banana."

= Text.Format( "#{0}, #{1} and #{2} are friends.",

{ "Rick", "Ben", "Mila" } )

// Returns values from a list: "Rick, Ben and Mila are friends."

// Adding a culture code allows you to force local formatting

= Text.Format( "Please pay us €#{0} before #{1}.",

{ 49.99, #date(2022, 5, 1 ) }, "EN-us" )

// Returns "Please pay us €49.99 before 5/1/2022."

= Text.Format( "Please pay us €#{0} before #{1}.",

{ 49.99, #date(2022, 5, 1 ) }, "NL-nl" )

// Returns "Please pay us €49,99 before 1-5-2022."


Противоположностью объединения значений является их разделение. В Power Query есть текстовые функции Text.Split и Text.SplitAny для разделения значений. В качестве аргументов функции принимают текстовое значение, а затем один или несколько разделителей в качестве второго аргумента. После выполнения функция возвращает список.

= Text.Split( "06-18-18-22", "1" ) // Returns { "06-", "8-", "8-22" }

= Text.Split( "06-18-18-22", "-" ) // Returns { "06", "18", "18", "22" }

= Text.SplitAny( "06-18-18-22", "-" ) // Returns { "06", "18", "18", "22" }

// For multiple separators, Text.SplitAny does not require a list.

= Text.SplitAny( "6|8|8-2", "-|" ) // Returns { "6", "8", "8", "2" }

= Text.SplitAny( "RickDeGroot", "er" ) // Returns { "RickD", "G", "oot" }


Очень похожа функция Text.ToList . Функция Text.ToList принимает строку и возвращает список, содержащий все односимвольные текстовые значения этой строки.

=Text.ToList( "Rick" ) // Returns { "R", "i", "c", "k" }


Другая категория функций сосредоточена на замене значений. Power Query предлагает функции Text.Replace и Text.ReplaceRange , каждая из которых имеет свой собственный подход к этому. Text.Replace заменяет все вхождения указанного текстового значения и заменяет их новым текстовым значением. Замена чувствительна к регистру .

= Text.Replace( "I enjoy sunny days", "enjoy", "love" ) // Returns "I love sunny days"

= Text.Replace( "my friend sees my show", "my", "his" ) // "his friend sees his show"


Text.ReplaceRange удаляет заданное количество символов и заменяет их предоставленным текстовым значением. Она работает очень похоже на функцию Text.RemoveRange , с той разницей, что она позволяет заменять удаленный диапазон предоставленным значением.

В качестве аргументов он принимает текстовое значение , смещение для начала замены, количество заменяемых значений и, наконец, заканчивается новым текстовым значением .

= Text.ReplaceRange( "RickVanGroot", 4, 3, "De" ) // Returns "RickDeGroot"

= Text.ReplaceRange( "RickVanGroot", 4, 3, "" ) // Returns "RickGroot"


Информационные функции

Информационные функции — это категория текстовых функций, которые анализируют текстовые значения по определенным критериям. Важная функция — Text.Contains . Функция Text.Contains проверяет, содержит ли текстовое значение строку. По умолчанию он учитывает регистр. Однако вы можете предоставить функции компаратор, чтобы изменить поведение и сделать сравнение без учета регистра.

= Text.Contains( "Red roses", "Red" ) // Returns true

= Text.Contains( "Red roses", "red" ) // Returns false

= Text.Contains( "Red roses", "red", Comparer.Ordinal ) // Returns false

= Text.Contains( "Red roses", "red", Comparer.OrdinalIgnoreCase ) // Returns true


Кроме того, есть функции, которые проверяют, начинается или заканчивается значение заданной строкой. Power Query использует для этого Text.StartWith и Text.EndsWith .

= Text.StartsWith( "I got the Power", "I got" ) // true

= Text.StartsWith( "I got the Power", "i got" ) // false

= Text.StartsWith( "I got the Power", "i got", Comparer.Ordinal ) // false

= Text.StartsWith( "I got the Power", "i got", Comparer.OrdinalIgnoreCase ) // true

= Text.EndsWith( "Biking to home", "home" ) // true

= Text.EndsWith( "Biking to home", "HOME" ) // false

= Text.EndsWith( "Biking to home", "HOME", Comparer.Ordinal) // false

= Text.EndsWith( "Biking to home", "HOME", Comparer.OrdinalIgnoreCase ) // true


Другие функции могут возвращать ценную информацию о значении. Text.Length возвращает длину текстового значения. Вы можете использовать эту информацию снова в других функциях, например, чтобы убедиться, что все символы имеют одинаковую длину.

= Text.Length( "Rick" ) // Returns 4

= Text.Length( "BIGorilla" ) // Returns 9


В своей самой простой форме функция Text.PositionOf возвращает первую позицию заданной подстроки в текстовом значении. Первый аргумент принимает текстовое значение , а второй требует подстроку . Если подстрока не найдена, функция возвращает -1 .

По умолчанию функция возвращает позицию первого вхождения подстроки. Однако вы можете использовать параметр вхождения в необязательном третьем аргументе, чтобы изменить это поведение. Вы можете использовать:

Occurrence.First (0) – возвращает позицию первого вхождения искомого значения
Occurrence.Last (1) – возвращает позицию последнего вхождения искомого значения
Occurrence.All (2) – возвращает список позиций всех вхождений искомого значения

= Text.PositionOf( "We all want pasta", "a" ) // Returns 3

// An index number determines which position of the value to return

// The first occurrence of a value is returned by default, returns 3

= Text.PositionOf( "We all want pasta", "a", Occurrence.First )

= Text.PositionOf( "We all want pasta", "a", 0 )

// Occurrence.Last returns the last occurrence, returns 16

= Text.PositionOf( "We all want pasta", "a", Occurrence.Last )

= Text.PositionOf( "We all want pasta", "a", 1 )

// Occurrence.All returns all occurrences, returns { 3, 8, 13, 16 }

= Text.PositionOf( "We all want pasta", "a", Occurrence.All )

= Text.PositionOf( "We all want pasta", "a", 2)

// You can also return the 2nd position from the start of the string, returns 8

Text.PositionOf( "We all want pasta", "a", Occurrence.All ) {1}

// By default the function searches capital sensitive. // Returns -1

= Text.PositionOf( "We all Want pasta", "w", 1 )

// Add an optional comparer to control the comparison. Returns 7

= Text.PositionOf( "We all Want pasta", "w", 1, Comparer.OrdinalIgnoreCase )


Кроме того, есть функция Text.PositionOfAny , которая возвращает позицию первого вхождения символов, указанных в списке. Функция принимает только список односимвольных строк. При желании вы можете указать параметр вхождения , чтобы указать, какую позицию вхождения нужно вернуть.

= Text.PositionOfAny( "You like guacemole", { "e" } ) // Returns 7

= Text.PositionOfAny( "You like guacemole", { "e", "u" } ) // Returns 2

// Occurrence parameters determine the position(s) to return

= Text.PositionOfAny( "You like guacemole", { "e", "u" }, Occurrence.First ) // 2

= Text.PositionOfAny( "You like guacemole", { "e", "u" }, Occurrence.Last ) // 17

= Text.PositionOfAny( "You like guacemole", { "e", "u" }, Occurrence.All )

// Returns { 2, 7, 10, 13, 17 }


Кодировать в другое значение

Вы можете использовать различные текстовые функции для преобразования значений из одного типа данных в другой.

= Text.From ( 123 ) // Returns "123"

// Tests if Power Query recognizes the string as a number

= Text.InferNumberType( "€11.11" ) // Returns number

// Returns DataFormat.Error: We couldn't match 'ABC' with any known number pattern.

= Text.InferNumberType( "ABC" )


В M различные функции используют значения символов Юникода . Чтобы узнать, какие символы представляют их, вы можете использовать функции Character.FromNumber и Character.ToNumber .

= Character.ToNumber( "!" ) // Returns 33

= Character.FromNumber( 33 ) // Returns "!"

= Character.FromNumber( 35 ) // Returns "#"

// You can use these characters to return a list of values.

// You can return the unicode characters representing 33-35:

= { "!" .. "#" } // Returns = { "!", """", "#" }
