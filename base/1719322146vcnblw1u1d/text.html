Использование DAX для управления диапазоном диаграммы в Power BI

Вы можете использовать выражения DAX для управления поведением некоторых аспектов визуальных элементов Power BI. В этой статье показано несколько примеров управления диапазоном диаграмм.

Марко Руссо и Альберто Феррари

ДАКС   POWER BI  

DAX — мощный инструмент в руках разработчика Power BI. Используя простые формулы DAX, вы можете не только вычислять интересные показатели, но и настраивать поведение визуальных элементов Power BI. В этой статье мы используем DAX для управления диапазоном диаграмм и получения более связных визуализаций.

Установите диапазон данных независимо от выбора среза

Начнем с пары простых примеров: первый показывает, как определить диапазон диаграммы независимо от выбора пользователя в срезах. Затем мы синхронизируем диапазоны двух диаграмм, которые разбивают данные по разным столбцам. Наконец, третий пример показывает, как использовать аналогичный метод для сравнения двух брендов, выбранных с помощью двух разных слайсеров. Этот последний сценарий основан на вопросе студента: в этом случае мы сначала быстро опишем неполное решение, не требующее использования DAX, затем немного подумаем и, наконец, создадим решение, требующее незначительных изменений модели вместе с некоторым DAX. код.

Первый пример отчета содержит один срез и одну гистограмму.

Здесь нет ничего особенного: срез выбирает одну или несколько категорий; диаграмма показывает объем продаж , разделенный по брендам. Однако обратите внимание на красный прямоугольник, который выделяет диапазон оси Y. Диапазон простирается от нуля до 3 млн, и он был автоматически вычислен Power BI, поскольку мы использовали Auto для минимальных и максимальных значений оси Y.

При каждом обновлении визуального элемента Power BI вычисляет значения, находит подходящий диапазон, а затем использует динамически вычисляемый диапазон для создания диаграммы. Функция работает плавно, но ее неудобство в том, что диапазон меняется в зависимости от выбора. Если мы выберем «Бытовая техника» в срезе «Категория», диапазон изменится от нуля до 1M.

Что, если мы хотим, чтобы диапазон оставался таким же, каким он был без выбора, от нуля до 3М? У нас есть два варианта: либо вручную вводим диапазон и получаем частичное решение, требующее обновления при изменении данных; или мы полагаемся на DAX для автоматического вычисления диапазона.

Мы можем создать показатель, который вычисляет максимальную сумму продаж по бренду независимо от выбора категории. Затем мы можем использовать эту меру для определения максимального значения оси Y:

Измерение в таблице продаж


Y-Axis range all categories =

    CALCULATE (

        ROUNDUP (

            MAXX (

                VALUES ( Product[Brand] ),

                [Sales Amount]

            ),

            -6

        ),

        REMOVEFILTERS ( Product[Category] )

    )

КОПИРОВАТЬКОНВЕНЦИИ№ 1

Здесь следует отметить пару интересных вещей. Во-первых, мы использовали ROUNDUP , чтобы получить диапазон порядка миллионов, используя шесть цифр слева от десятичной точки. Если бы мы этого не сделали, в диапазоне использовалось бы какое-то странное число, например 2,87M, что не очень удобно для пользователя. Во-вторых, мы использовали CALCULATE , чтобы удалить фильтр только для Product[Category] . В зависимости от требований вы можете использовать другой контекст фильтра для получения требуемого диапазона.

Используя эту меру как выражение максимального Y-диапазона, масштаб отчета не меняется даже при наличии фильтра среза.

Синхронизировать диапазон данных на двух графиках

Первый сценарий был намеренно простым; легкая разминка. Второй сценарий включает в себя две диаграммы: одну по брендам и одну по категориям .

Два графика расположены таким образом, чтобы читателю было естественно сравнивать высоту столбцов. К сожалению, поскольку диапазон двух визуальных элементов вычисляется независимо, они не имеют одного и того же диапазона по оси Y. Один идет до 3М, другой до 5М.

Мы можем использовать технику, использованную в предыдущем примере. Однако на этот раз нам нужно найти максимум два набора: один — набор брендов, другой — набор категорий:

Измерение в таблице продаж


Y axis range Brand and Category =

    VAR MaxRangeByBrand =

        ROUNDUP (

            MAXX (

                VALUES ( Product[Brand] ),

                [Sales Amount]

            ),

            -6

        )

    VAR MaxRangeByCategory =

        ROUNDUP (

            MAXX (

                VALUES ( Product[Category] ),

                [Sales Amount]

            ),

            -6

        )

    VAR Result = MAX ( MaxRangeByBrand, MaxRangeByCategory )

    RETURN Result

КОПИРОВАТЬКОНВЕНЦИИ# 2

Код прост. По сути, он дублирует ту же логику, что и раньше, а затем вычисляет максимум из двух переменных. Как только этот максимум используется в качестве максимального значения оси Y на обеих диаграммах, столбцы можно сравнивать визуально.

Синхронизировать диапазон данных для двух разных вариантов выбора

Последний сценарий, который мы хотим решить, был предложен студентом, вдохновившим на создание этой статьи. Цель состоит в том, чтобы сравнить годовые продажи двух произвольных брендов, используя две гистограммы. Мы можем получить следующий отчет, создав два среза, две диаграммы, а затем тщательно настроив визуальные взаимодействия так, чтобы каждый срез работал на своей собственной диаграмме.

Несмотря на эту работу, две диаграммы невозможно сравнить визуально, поскольку их оси Y не показывают одинаковый диапазон. Мы хотели бы получить максимальные годовые продажи Fabrikam и AdventureWorks, а затем использовать это значение в качестве верхнего предела для обоих визуальных эффектов.

К сожалению, на этот раз сценарий решить труднее. Изображение слева фильтрует Adventure Works и не имеет ни малейшего представления о том, что Fabrikam выбрана в изображении справа. Это связано с тем, как мы устанавливаем визуальные взаимодействия. Мы должны включить визуальное взаимодействие между срезом справа и столбчатой ​​диаграммой слева, чтобы распространить фильтр справа на визуал слева. Однако, сделав это, мы помешаем работе отчета, поскольку два среза будут фильтровать разные бренды, что приведет к пустой выборке.

Для решения требуются две дополнительные таблицы. Мы хотим, чтобы фильтр обоих срезов достигал обоих визуальных элементов; однако мы хотим контролировать, как фильтр будет работать как с отображаемым значением, так и с расчетом диапазона по оси Y.

Сначала мы создаем две дополнительные таблицы, не связанные с моделью, которые содержат значения брендов:

Расчетная таблица


1

Brands 01 = ALLNOBLANKROW ( Product[Brand] )

КОПИРОВАТЬКОНВЕНЦИИ№ 3

Расчетная таблица


1

Brands 02 = ALLNOBLANKROW ( Product[Brand] )

КОПИРОВАТЬКОНВЕНЦИИ# 4

Эти две таблицы не имеют связи с другими таблицами модели.

Затем мы используем две таблицы в качестве источников для срезов. При этом столбчатые диаграммы перестают иметь смысл, поскольку в Sales отсутствует активный фильтр . Таким образом, значения на диаграммах всегда вычисляют объем продаж для всех брендов.

Следующий шаг — восстановить визуальные взаимодействия, чтобы оба слайсера работали с обоими визуальными элементами. Мы создаем две меры, которые считывают содержимое правильного среза и вычисляют значение для диаграммы:

Использование этих двух показателей возвращает нас к визуализации, которая у нас была вначале, с той заметной разницей, что все визуальные взаимодействия теперь активны. Таким образом, на обеих диаграммах у нас есть возможность прочитать содержимое обоих выбранных срезов.

Теперь мы можем создать меру, которая считывает содержимое обоих срезов и вычисляет максимальные годовые продажи обоих брендов:

Y-axis range both brands =

    VAR Range01 = MAXX ( VALUES ( 'Date'[Year] ), [Sales 01] )

    VAR Range02 = MAXX ( VALUES ( 'Date'[Year] ), [Sales 02] )

    VAR Result = MAX ( Range01, Range02 )

    RETURN

        Result

Используя эту меру в качестве максимального диапазона оси Y, мы получаем желаемый отчет.

Оба визуальных элемента теперь показывают один и тот же диапазон, и пользователи могут визуально сравнивать столбцы.

Заключение

Как вы видели, DAX — жизненно важная часть Power BI. Это не только полезно для вычисления значений. Проявив творческий подход и обладая базовыми навыками моделирования данных, вы можете получить красивые и последовательные визуализации.
