Вам нужно выяснить, сколько рабочих дней между двумя датами в Power Query? В этой статье я покажу вам шаг за шагом, как вычислить разницу между двумя датами. Я объясню, как можно использовать Power Query для подсчета дней и легкого исключения выходных и праздничных дней. Это похоже на подсчет дней в календаре, только более точно.

И если в Excel и SQL есть такая функция, то в языке M ее нет. Итак, эта статья помогает создать функцию DATEDIFF в Power Query. Имке сделал что-то подобное здесь, но мне хотелось, чтобы функция «Рабочие дни» работала с отрицательными приращениями. Поэтому в этом посте показан скорректированный подход.

К концу этой статьи вы сможете быстро узнать, сколько рабочих дней проходит между любыми двумя датами, и включить номер рабочего дня в свой календарь . Давайте начнем и научимся рассчитывать рабочие дни в Power Query!

Функция логики рабочих дней

Чтобы создать функцию NetWorkDays в Power Query, вам нужно составить список рабочих функций. Чтобы улучшить свои навыки Power Query, я всегда советую людям освоить функции списков . Они полезны во многих сценариях и часто поддерживают мощные решения.

Допустим, вы хотите найти количество рабочих дней с 1 октября 2022 года по 31 декабря 2022 года. А 26 декабря 2022 года считается выходным днем.

Необходимые шаги для прибытия туда:

  1. Рассчитайте количество рабочих дней от даты начала до даты окончания включительно.
  2. Удалить даты выходных
  3. При необходимости удалите указанные праздничные даты.
  4. Посчитайте количество оставшихся свиданий

Вы можете пойти дальше и использовать что-то вроде:

let

// Define start and ending dates

DateStart = #date( 2022, 10, 1 ),

DateEnd = #date( 2022, 12, 31 ),

// Find the number of days between Start and End Date

DaysBetween = Duration.Days( DateEnd - DateStart ),

// The number of days should create a series that begins at the start

// date and ends at the ending date. You need an additional day here.

NumberOfDays = DaysBetween + ( if DateEnd < DateStart then - 1 else + 1 ),

// Direction indicates date series direction (positive or negative)

Direction = Number.Sign( NumberOfDays ),

// Creates dates between Start and End Date. Series can increment

// or decrement with each step, decided by the Direction variable.

ListOfDates =

List.Dates(

DateStart,

Number.Abs( NumberOfDays ),

#duration( Direction, 0, 0, 0 ) // positive or negative steps

),

ListOfWeekDays =

List.Select(

ListOfDates, // exclude weekends

each Date.DayOfWeek( _, Day.Monday ) + 1 <= 5

),

// List of holidays to exclude

Holidays = { #date(2022, 12, 26) },

// Selects only the non-holiday dates.

ListOfWorkDays =

List.Difference(

ListOfWeekDays,

Holidays ?? {} // Remove Holidays. When blank, remove nothing

),

// Counts the number of working days

NumberOfWorkDays = List.Count( ListOfWorkDays ),

// Multiplies the number of working days by 1 or -1, depending on

// whether starting date is before or after ending date.

Positive_or_negative_WorkingDays = NumberOfWorkDays * Direction

in

Positive_or_negative_WorkingDays


Перед сценарием стоит проработанная версия, которая хорошо работает для двух осязаемых свиданий. Чтобы сделать работу более гибкой, вы можете включить некоторую проверку ошибок. В финальный сценарий внесены небольшие изменения.

В первой строке указываются параметры функции. Это позволяет нам легко вызывать логику рабочего дня как пользовательскую функцию.

Следующее отличие заключается в том, что если пользователь не заполняет дату начала или дату окончания, функция по умолчанию будет использовать 1 января 1900 года. Для этого мы используем оператор COALESCE в M.

Аналогично, если список праздников остается пустым, функция ниже использует пустой список. И здесь на помощь приходит оператор COALESCE.

Окончательный код рабочего дня для функции:

let func =

(StartDate, EndDate, optional Holidays as list ) =>

let

// When a date is null value, start calculating from 1 Jan 1900

DateStart = StartDate ?? #date( 1900, 1, 1 ),

DateEnd = EndDate ?? #date( 1900, 1, 1 ),

// Find the number of days between Start and End Date

DaysBetween = Duration.Days( DateEnd - DateStart ) ,

// The correct date series requires the number of days FROM start to end,

// not the number of days BETWEEN start and end.

NumberOfDays = DaysBetween + (if DateEnd < DateStart then -1 else + 1 ),

// Direction indicates date series direction (positive or negative)

Direction = Number.Sign( NumberOfDays ),

Result =

List.Count(

List.Difference(

List.Select(

List.Dates(

DateStart,

Number.Abs( NumberOfDays ),

#duration( Direction, 0, 0, 0) // positive or negative steps

),

each Date.DayOfWeek( _, Day.Monday ) + 1 <= 5 // excl weekends

),

Holidays ?? {} // exclude holidays. If omitted, use empty list

)

)

* Direction // Show number of days positive or negatively

in Result,

Result ",

"]}]

Примеры кода рабочих дней

Итак, как лучше всего использовать эту функцию?

Рабочий день Номер недели/месяца/года

Эта функция может быть полезна при создании календарей. Например, вы можете добавить поле, показывающее номер рабочего дня недели.

Если вы сначала добавите столбец с датой начала недели, вы сможете затем сослаться на него и вернуть номер рабочего дня недели:

= Date.StartOfWeek( [Date] ) // Add Start of Week date

= fxNetWorkDay( [Start of Week], [Date], { #date( 2022, 12, 26 ) } )

// Returns a column with the Workday number of the week


Аналогичным образом вы можете добавить дату начала месяца или начала года. С их помощью вы сможете вычислить номер рабочего дня месяца или года.

= Date.StartOfMonth( [Date] ) // Add Start of Month date

= fxNetWorkDay( [Start of Month], [Date], { #date( 2022, 12, 26 ) } )

// Returns a column with the Workday number of the month.

= Date.StartOfYear( [Date] ) // Add Start of Year date

= fxNetWorkDay( [Start of Year], [Date], { #date( 2022, 12, 26 ) } )

// Returns a column with the Workday number of the year.


Добавить праздничные даты

При некоторых расчетах чистого рабочего дня важно учитывать список праздников. Вы можете легко настроить эту функцию, чтобы учесть желаемый список праздников.

Есть несколько способов сделать это.

Даты жесткого кодирования

Если у вас есть короткий список праздников, вы можете жестко запрограммировать его в своей формуле.

= fxNetWorkDay( [StartDate],

[EndDate],

{ #date( 2022, 12, 26 ) } )

// Counts the number of days between the starting and ending date

// taking into account 26 December 2022 as holiday

= fxNetWorkDay( [StartDate],

[EndDate],

{ #date( 2022, 12, 26 ), #date( 2022, 12, 27 ), #date( 2022, 12, 28 ) }

)

// Counts the number of days between the starting and ending date

// taking into account 26, 27 and 28 December 2022 as holiday


Этот метод удобен только при небольшом количестве дат. В большинстве ситуаций вам понадобится ссылка на таблицу с праздниками или на список праздников.

Праздничный стол

В большинстве ситуаций удобен праздничный стол. Если у вас есть таблица праздников под названием Holidays и она содержит столбец с именем Date , вы можете сослаться на свои праздники, написав:

= fxNetWorkDay( [StartDate],

[EndDate],

Holidays[Date] )

// Respects holidays from the 'Date' column in the 'Holidays' table


Список праздников

В других случаях у вас может быть список праздников. Либо в том же запросе, либо, возможно, вы использовали функцию детализации Power Query, чтобы увеличить масштаб столбца таблицы с праздниками. В любом случае вы можете легко ссылаться на свой список праздников, указав имя запроса со списком праздников или имя шага со списком праздников.

= fxNetWorkDay( [StartDate],

[EndDate],

HolidaysList ) // Can be a separate query or a an earlier step name.

// Respects Holidays from the HolidaysList


Заключение

Вот и все! Теперь вы знаете, как рассчитать количество рабочих дней между двумя датами в Power Query. Вы создали функцию DATEDIFF в M. Ее легко использовать в обычной таблице дат , и она помогает получить точное количество дней, которое вам нужно. Комбинируя функции списка, включая проверку ошибок и превращая логику в функцию, теперь вы можете легко найти количество рабочих дней между двумя днями.

Если вам это понравилось, возможно, вам будет интересно узнать, как вернуть N-й рабочий день, начиная с заданной даты .
