Я нашел этот фрагмент как альтернативное решение. Это более изящное удаление библиотек, чем переделка virtualenv:

pip freeze | xargs pip uninstall -y

Если у вас есть пакеты, установленные через VCS, вам необходимо исключить эти строки и удалить пакеты вручную (см. комментарии ниже):

pip freeze --exclude-editable | xargs pip uninstall -y

Если у вас есть пакеты, установленные непосредственно с github/gitlab, они будут иметь расширение @. Нравиться:

django @ git+https://github.com/django.git@<sha>

Вы можете добавить cut -d "@" -f1, чтобы получить только имя пакета, необходимое для его удаления.

pip freeze | cut -d "@" -f1 | xargs pip uninstall -y


Это будет работать для всех систем Mac, Windows и Linux. Чтобы получить список всех пакетов pip в файле require.txt (Примечание. Это приведет к перезаписи файла require.txt, если он существует, иначе будет создан новый, а также, если вы не хотите заменять старый файл require.txt, укажите другое имя файла. во всех последующих командах вместо файла require.txt).

pip freeze > requirements.txt

Теперь удаляю по одному

pip uninstall -r requirements.txt

Если мы хотим удалить все сразу, то

pip uninstall -r requirements.txt -y

Если вы работаете над существующим проектом, в котором есть requirements.txtфайл, а ваша среда отличается, просто замените requirements.txtв приведенных выше примерах на toberemoved.txt. Затем, выполнив описанные выше шаги, вы можете использовать , requirements.txtчтобы обновить теперь чистую среду.

Бонус: псевдоним bash

Для удобства я добавляю это в свой профиль bash:

alias pipuninstallall="pip uninstall -y -r <(pip freeze)"

Затем запустите:

pipuninstallall

Альтернатива для Pipenv

Если вы используете Pipenv , вы можете запустить:

pipenv uninstall --all

Альтернатива поэзии

Если вы используете Poetry , запустите:

poetry env remove --python3.9

(Обратите внимание, что вам нужно изменить номер версии, чтобы он соответствовал вашей версии Python.)

Мне удалось это сделать, выполнив следующие действия:

  1. Создайте файл требований reqs.txtсо списком установленных пакетов.

pip freeze > reqs.txt

  1. Затем удалите все пакеты изreqs.txt

# -y means remove the package without prompting for confirmationpip uninstall -y -r reqs.txt

Мне нравится этот метод, поскольку у вас всегда есть файл требований к протоколу, к которому можно обратиться, если вы допустите ошибку. Он также воспроизводим и кроссплатформен (Windows, Linux, MacOs).

В Windows, если вы pathнастроены правильно, вы можете использовать:

pip freeze > unins && pip uninstall -y -r unins && del unins

Аналогичный случай должен быть и для Unix-подобных систем:

pip freeze > unins && pip uninstall -y -r unins && rm unins

Просто предупреждение о том, что это не совсем надежно, поскольку вы можете столкнуться с такими проблемами, как «Файл не найден», но, тем не менее, в некоторых случаях это может сработать.

РЕДАКТИРОВАТЬ: Для ясности: uninsэто произвольный файл, в который записываются данные при выполнении этой команды:pip freeze > unins

Этот файл, который он, в свою очередь, записывает, затем используется для удаления вышеупомянутых пакетов с подразумеваемым согласием/предварительным одобрением черезpip uninstall -y -r unins

После завершения файл окончательно удаляется.
