Мы все были там. Наш код выполняет задание, и это займет некоторое время. Почесывая голову, мы молимся, чтобы наш код не застрял в каком-то бесконечном цикле. При работе с большими наборами данных даже самые простые операции могут занять часы.

К счастью, существуют библиотеки, которые предоставляют эту функциональность, используя всего пару строк. В этом уроке я представлю четыре таких библиотеки и сравню их на основе множества факторов.

# Прогресс

Progress - довольно простая и интуитивно понятная библиотека. Он предоставляет широкий выбор стилей на выбор. Все индикаторы выполнения могут быть интегрированы в наши циклы с помощью диспетчера контекста или путем включения итерируемого объекта в метод. Давайте посмотрим на некоторые из них.

Простой индикатор выполнения, заполненный хешем.


from time import sleep

from progress.bar import Bar

with Bar('Processing...') as bar:

for i in range(100):

sleep(0.02)

bar.next()

Сначала мы импортируем класс Bar из модуля progress.bar и создаем его объект. Мы предоставляем префикс 'Processing...', который будет добавлен в начало нашего индикатора выполнения. Чтобы обновить индикатор выполнения, мы используем метод next() в конце каждой итерации.

Нам доступно несколько аргументов, например fill и suffix. Подробности о них можно найти здесь. Давайте попробуем изменить заливку с # на @, а также посмотрим на ETA.


from time import sleep

from progress.bar import Bar

with Bar('Loading', fill='@', suffix='%(percent).1f%% - %(eta)ds') as bar:

for i in range(100):

sleep(0.02)

bar.next()

Бывают случаи, когда мы не знаем, сколько времени займет операция. В таких случаях мы можем использовать счетчики для визуализации обработки, а не индикаторы выполнения.


from time import sleep

from progress.spinner import MoonSpinner

with MoonSpinner('Processing…') as bar:

for i in range(100):

sleep(0.02)

bar.next()

# Progressbar2

Python-Progressbar (также известный как Progressbar2) - чрезвычайно популярный и простой в использовании модуль. Он предоставляет некоторые очень мощные функции, такие как автоматическое изменение размера, если это поддерживает система. Вы можете управлять форматом индикатора выполнения в виде виджетов, таких как AbsoluteETA, AdaptiveETA и т. Д.

Давайте посмотрим на базовый вариант использования:


from time import sleep

from progressbar import progressbar

for i in progressbar(range(100)):

sleep(0.02)

Примечание. Хотя вначале мы установили библиотеку как progressbar2, она будет импортирована как progressbar.

Объединение индикаторов выполнения с выводом на печать

Одна вещь, которую мы раньше не обсуждали с библиотекой прогресса: Что происходит, когда цикл for содержит функцию печати?
Посмотрим:


from time import sleep

from progress.bar import Bar

with Bar('Processing',max = 5) as bar:

for i in range(5):

sleep(0.1)

print('\n',i)

bar.next()

view rawprogress_5.py hosted with ❤ by GitHub

Как видите, это сделает индикатор выполнения очень беспорядочным, и, к сожалению, в библиотеке выполнения для этого нет обходного пути. Однако Progressbar2 позволяет перенаправлять стандартный вывод.


from time import sleep

from progressbar import progressbar

for i in progressbar(range(100), redirect_stdout=True):

print('Some text', i)

sleep(0.1)

С точки зрения кода это очень похоже. Мы представим индикатор выполнения по умолчанию, а затем еще один с парой модификаций:

from time import sleep

# TQDM

Если вам нужен быстрый, но мощный индикатор прогресса без всякой ерунды, не смотрите дальше.

TQDM предоставляет быструю структуру с большим количеством настроек и информации. Даже со всеми функциями его очень просто настроить. Взглянем:


from tqdm import tqdm

from time import sleep

for i in tqdm(range(100)):

sleep(0.02)

Заключительные слова

Индикаторы прогресса предоставляют нам:

  1. Визуальный сигнал о том, что обработка продолжается.
  2. Надежная оценка времени выполнения кода. (Чрезвычайно важно при обучении на больших наборах данных).
  3. Индикация того, что программа зависла из-за какой-либо ошибки и должна быть остановлена.

TQDM имеет самые низкие накладные расходы из всех рассмотренных выше библиотек - около 60 нс на итерацию. Для сравнения, хорошо зарекомендовавший себя ProgressBar2 имеет накладные расходы 800 нс / л.

Большинство этих библиотек не имеют зависимостей.

Progressbar2 и alive-progress официально поддерживают операторы печати вместе с индикатором выполнения.
