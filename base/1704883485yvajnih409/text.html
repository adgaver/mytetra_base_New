Важность удаления дубликатов

Удаление повторяющихся строк — важный шаг при очистке данных в Power Query. Дубликаты могут вызвать проблемы во многих ситуациях. Например, когда:

  • Выполнение соединения: дубликаты в объединяемой таблице приводят к дублированию строк в целевой таблице.
  • Создание таблицы измерений: невозможно создать связь «один ко многим» с дубликатами на одной стороне.
  • Проверка значений: дубликаты могут указывать на то, что в более раннем объединении были повторяющиеся значения.

Чтобы предотвратить это, нам нужно выражение для проверки дубликатов и фильтрации нашего набора данных только для включения уникальных значений.

Основы удаления дубликатов

Итак, как удалить дубликаты с помощью редактора Power Query? Внутри операция использует функцию Table.Distinct для удаления дубликатов. Синтаксис функции следующий:

= Table.Distinct(

table as table,

optional equationCriteria as any

)


У этой функции есть два аргумента: первый аргумент — это таблица, из которой нужно удалить дубликаты , а второй аргумент — необязательный методуравнения, который сообщает функции, какие столбцы следует просматривать при удалении дубликатов . Вы можете указать имя столбца или указать функции учитывать или игнорировать заглавные буквы. Если вы опустите уравнениеCriteria, функция удалит дубликаты из всей таблицы.

Удаление дубликатов из одного столбца

Начнем с удаления дубликатов из одного столбца. Это один из самых простых методов удаления дубликатов в Power Query, и его действительно легко сделать.

Чтобы удалить дубликаты из одного столбца, вы:

  • Щелкните правой кнопкой мыши столбец, содержащий дубликаты.
  • Выберите опцию « Удалить дубликаты ».
  • Теперь нажмите ОК.

Вот и все! Power Query удалит все дубликаты из выбранного столбца.

Операция удаления дубликатов использует функцию Table.Distinct в Power Query. При выполнении операции над таблицей с одним столбцом приведенная выше операция покажет:

= Table.Distinct( Source )


Это удалит дубликаты из всей таблицы. Если в вашей таблице несколько столбцов, приведенный ниже код удаляет дубликаты только из столбца « Фрукты ».

= Table.Distinct( Source, { "Fruit" } )


Но что, если у вас есть повторяющиеся значения в нескольких столбцах? Без проблем!

Удаление дубликатов на основе нескольких столбцов

Самый простой способ удалить дубликаты на основе нескольких столбцов:

  • Удерживайте клавишу CTRL.
  • Выберите столбцы, из которых вы хотите удалить дубликаты.
  • Нажмите «Удалить дубликаты» .

Теперь Power Query удалит все строки с повторяющимися значениями в выбранных столбцах. Эта операция направлена ​​на возврат уникальных комбинаций значений.

= Table.Distinct(

Source,

{ "Brand", "Country"} )


При выполнении этой операции по-прежнему используется функция Table.Distinct . На этот раз второй аргумент содержит имена обоих столбцов в списке. Если вам не помешало бы освежить знания, взгляните на варианты использования и синтаксис списков .

Итак, вы знаете, как удалить дубликаты из одного или нескольких столбцов, но что, если вы хотите удалить дубликаты без учета регистра?

Удаление дубликатов без учета регистра

Чтобы удалить дубликаты без учета регистра, вы можете использовать необязательный аргумент уравнения Table.Distinct . Независимо от того, работаете ли вы с одним или несколькими столбцами или даже со всей таблицей, вы можете легко удалить дубликаты без учета регистра.

Итак, что, если у вас есть набор данных, содержащий дубликаты, и вы хотите выполнить операцию удаления дубликатов с учетом регистра в столбце Wood ? Удаление дубликатов (с учетом регистра) удаляет только строку со значением « Кедр ».

Для этого вы можете использовать функцию Table.Distinct и указать имя столбца «Дерево» во втором аргументе. Результатом является удаление дубликатов с учетом регистра, как показано ниже.

Теперь предположим, что вы хотите игнорировать заглавные буквы при удалении дубликатов . Самый простой способ удалить дубликаты без учета регистра — добавить функцию сравнения :

  • Щелкните правой кнопкой мыши столбец, содержащий дубликаты.
  • Выберите «Удалить дубликаты».
  • Укажите имя столбца и Comparer.OrdinalIgnoreCase в качестве критериев уравнения.
  • Используйте формат: { «Имя столбца», Comparer.OrdinalIgnoreCase }

Результатом является удаление дубликатов без учета регистра.

= Table.Distinct(

Source,

{ "Wood", Comparer.OrdinalIgnoreCase } )


Но что, если вы хотите применить этот метод к нескольким столбцам?

= Table.Distinct(

Source,

{ { "Wood", Comparer.OrdinalIgnoreCase },

{ "Area", Comparer.OrdinalIgnoreCase } } )


Вы можете использовать функцию Table.Distinct и предоставить метод для поиска дубликатов для каждого столбца. Это позволяет вам указать, следует ли игнорировать заглавные буквы ( Comparer.OrdinalIgnoreCase ) или учитывать заглавные буквы ( Comparer.Ordinal ) для каждого столбца индивидуально.

Итак, вот и все — несколько основных методов удаления дубликатов в Power Query. Независимо от того, работаете ли вы с одним столбцом, с несколькими столбцами или проверяете чувствительность к заглавным буквам, Power Query позволяет легко избавиться от дубликатов и привести данные в превосходную форму.

Но имейте в виду, что методы, показанные до сих пор, не всегда возвращают одни и те же результаты. Давайте посмотрим, почему это так, в следующем разделе.

Непоследовательные результаты

Знаете ли вы, что операция удаления дубликатов в Power Query часто возвращает противоречивые результаты? Это может показаться странным, но на самом деле это побочный эффект некоторых оптимизаций, которые Power Query использует для достижения максимальной эффективности.

Одной из таких оптимизаций является Query Folding, которая отправляет определенные операции для обработки в источник данных, а не в Power Query . Хотя это может помочь сделать работу быстрее и эффективнее, это также может вызвать проблемы при удалении дубликатов. Без использования дополнительных мер нет гарантии, что удаление дубликатов вернет те же записи . Это означает, что вы не можете предполагать, что удаление дубликатов вернет только первую повторяющуюся строку в вашем наборе данных.

Но есть простое решение этой проблемы случайности. Чтобы сделать операцию удаления дубликатов предсказуемой, вы можете использовать Table.Buffer . Эта функция заставляет Power Query кэшировать данные в памяти, а это означает, что вы можете контролировать, какие записи возвращаются при удалении дубликатов.

Итак, если вы хотите быть уверены, что получаете согласованные результаты при удалении дубликатов, обязательно используйте Table.Buffer . В следующей главе мы углубимся в эту тему и рассмотрим продвинутые методы.

Передовые методы удаления дубликатов

Иногда удаление дубликатов не так просто, как кажется, и для выполнения этой работы могут потребоваться более сложные методы. Вот здесь и начинается эта глава — мы собираемся изучить некоторые продвинутые методы удаления дубликатов в Power Query.

В этой главе мы углубимся в то, как можно удалить дубликаты на основе условия . Мы рассмотрим получение первого или последнего значения, самого раннего или самого последнего значения и самого низкого или самого высокого значения.

Сохранять первое или последнее значение

Сначала давайте поговорим о том, как удалить дубликаты и сохранить первое или последнее значение в наборе данных. По умолчанию Table.Distinct возвращает первую найденную повторяющуюся строку . Но, как мы узнали из предыдущей главы, порядок сортировки вашей таблицы не гарантирован, что может вызвать проблемы.

Чтобы гарантировать сохранение первого или последнего значения, нам нужно отсортировать наши данные и убедиться, что они остаются отсортированными.

Чтобы удалить дубликаты и получить первое значение в наборе данных, вы:

  • Сортируйте свои данные
  • Выберите столбцы, из которых нужно удалить дубликаты
  • Оберните данные в Table.Buffer.
  • Удалить дубликаты

Вы можете сделать это с помощью этого набора данных.

В конечном итоге вы удалите дубликаты, но сохраните первый экземпляр в своей категории. Итак, чтобы вернуть первое значение при удалении дубликатов, вы сортируете свой код с помощью:

= Table.Buffer(

Table.Sort( Dataset,

{ {"Category", Order.Ascending },

{"Description", Order.Ascending } } ) )


Если вместо этого вы хотите сохранить последнее значение, вы можете просто отсортировать данные в порядке убывания по столбцу, в котором вы хотите сохранить последнее значение. Затем оберните данные в Table.Buffer и удалите дубликаты.

= Table.Buffer(

Table.Sort( Dataset,

{ {"Category", Order.Ascending },

{"Description", Order.Descending } } ) )


Сохранять самое раннее или последнее значение

Если вы хотите вернуть самое раннее или самое последнее значение, вы можете сделать что-то подобное. Чтобы вернуть уникальные строки для категории с самой последней датой, вы:

  1. Отсортируйте данные по категориям и в порядке убывания для столбца даты.
  2. Удалите дубликаты из соответствующих столбцов.

// Sorts and buffers data, so removing duplicates returns the latest category row

= Table.Buffer(

Table.Sort( Dataset,

{ {"Category", Order.Ascending },

{"Date", Order.Descending } } ) )

// Sorts and buffers data, so removing duplicates returns the earliest category row

= Table.Buffer(

Table.Sort( Dataset,

{ {"Category", Order.Ascending },

{"Date", Order.Ascending } } ) )


Сохраняйте самое низкое или максимальное значение

Последний расширенный метод возвращает наименьшее или наибольшее значение в группе . Для достижения этой цели мы могли бы использовать описанный выше метод. Тем не менее, поскольку я уверен, что вы знаете, как сортировать данные аналогичным образом, давайте рассмотрим альтернативу.

Чтобы удалить дубликаты и вернуть наименьшее значение, вы можете:

  1. Сгруппируйте данные.
  2. Выполните операцию «Все строки».
  3. Настройте M-код, чтобы вернуть строку с наименьшим значением.
  4. Разверните столбец записи, чтобы получить результат.

Давайте посмотрим, как это может работать. Операция «Группировать по» возвращает только уникальные строки для выбранных столбцов.

Заключение

И здесь у вас есть все, что вам нужно знать об удалении дубликатов в Power Query. Независимо от того, имеете ли вы дело с одним или несколькими столбцами, дубликатами с учетом или без учета регистра или с расширенными методами сохранения первого, последнего, самого раннего, самого последнего, самого низкого или самого высокого значения, Power Query поможет вам.

Помните, что удаление дубликатов — важный шаг в очистке ваших данных. Удалив дубликаты, вы сможете избежать проблем с объединениями, таблицами измерений и подготовить данные для вашей модели данных.

Просто помните о проблеме случайности, которая может возникнуть при использовании функции Table.Distinct , и используйте Table.Buffer для обеспечения согласованных результатов.
