Оператор IN в M/Power Query

Рик — основатель BI Gorilla. Он считает, что обучение — одно из величайших удовольствий в жизни, и делится своими знаниями, чтобы помочь вам улучшить свои навыки.

Вы пытаетесь использовать оператор IN в Power Query, но не знаете, как это сделать? Оператор IN не встроен в Power Query, но мы все равно можем добиться того же, используя другие функции. В этой статье я покажу вам, как использовать оператор IN в Power Query .

Введение

При создании условных операторов if общим требованием является проверка, равно ли значение одному из многих. В этом операторе вы можете объединить несколько операторов ИЛИ.

В большинстве языков программирования для этого используется оператор IN. Оператор IN призван упростить условия, чтобы можно было легко предоставить список значений для проверки.

Допустим, у вас есть компания, которая продает одежду. В настоящее время у вас есть скидка на джинсы, юбки и колготки. Вы хотите классифицировать их по «Распродаже» и «Обычным». Вы можете классифицировать их, создав оператор if:

if [Product] = "Jeans" or [Product] = "Skirts" or [Product] = "Tights"

then "Sale"

else "Regular"


Этот оператор работает хорошо и подробно возвращает правильные результаты. Мы проиллюстрировали это всего на трех продуктах, но представьте, что вы проделали то же самое для 15 или более продуктов.

Помимо того, что набирать список товаров вручную очень неудобно, это еще и требует большого количества ненужного повторения кода. В этом случае столбец [Продукт] повторяется для каждого товара.

Репликация оператора IN

Итак, какой код можно использовать для репликации оператора IN? Вы можете переписать приведенное выше утверждение так:

if List.Contains( { "Jeans", "Skirts", "Tights" }, [Product] )

then "Sale"

else "Regular"


Таким образом, функция List.Contains копирует оператор IN в M. Она проверяет, соответствует ли значение в столбце [Продукт] одному из значений в списке.

Вы также можете объединить эту конструкцию и проверить несколько условий ИЛИ одновременно.

if List.Contains( { "Jeans", "Skirts", "Tights" }, [Product] )

and List.Contains( { "Blue", "Red", "Yellow" }, [Color] )

then "Sale"

else "Regular"


Чтобы отметить все продукты, кроме тех, которые есть в вашем списке, вы можете обратить результат, используя слово not:

if not List.Contains( { "Jeans", "Skirts", "Tights" }, [Product] )

then "Sale"

else "Regular"


Оператор IN с несколькими столбцами

Вы также можете реплицировать оператор IN в нескольких столбцах. Это может быть полезно, если вы хотите проверить, существует ли в таблице комбинация значений. В примерах используется функция Table.Contains , она предполагает работу с таблицами и записями и может выглядеть следующим образом:

= List.Contains(

{ {"Jeans", "Blue"},

{"Skirt", "Yellow"} },

{ [Product], [Color] } // Turns Columns into list

)

= Table.Contains(

#table( // creates table

{"Product", "Color"}, // with column names

{ {"Jeans", "Blue"}, // values 1st row

{"Skirt", "Yellow"} } ), // values 2nd row

[[Product], [Color]] // checks if values exist in table

)

= Table.Contains(

Table.FromRecords(

{ [Product = "Jeans", Color = "Blue"],

[Product = "Skirt", Color = "Yellow"] },

type table [Product = Text.Type, Color = Text.Type]

),

[[Product], [Color]]

)


Это были всего лишь три примера. Существует несколько других способов построения таблиц, которые приводят к аналогичным результатам. В следующей статье вы можете узнать больше о создании таблиц в Power Query .

Заключение

В этой статье показано, как можно воспроизвести оператор IN на языке Power Query M. Оператор IN, возможно, еще не встроен в Power Query, но с другими функциями мы можем добиться тех же результатов.

Это делает ваш код намного короче при проверке наличия значения в списке. Это полезно при проверке одного столбца, но также можно работать и с несколькими столбцами.
