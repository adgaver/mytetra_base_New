Написано Томом (AnalystCave)  в3 ноября 2018 г. в экселе , МС офис

Чтобы удалить файл с помощью VBA, вам нужно использовать функцию уничтожения VBA . Однако проблема с функцией VBA Kill заключается в том, что она не проверяет, существует ли файл в первую очередь, а вместо этого выдает ошибку.

filePath = "C:\pathtoyourfile\file"

 

'This might throw an error in case file does not exist

Kill filePath

 

'This will never throw an error

If Len(Dir(filePath)) > 0 Then

   SetAttr filePath, vbNormal

   Kill filePath

End If

VBA УДАЛЕНИЕ НЕСКОЛЬКИХ ФАЙЛОВ С ИСПОЛЬЗОВАНИЕМ ПОДСТАНОВОЧНЫХ ЗНАКОВ (*)

Если у вас есть много файлов для удаления в одном каталоге, которые соответствуют определенной последовательности символов в имени, вы можете использовать подстановочные знаки (*).

Подстановочный знак (*) заменяет любое количество символов (более одного) в имени файла. Пример ниже:

Kill "my_*.xlsx"

 

'will delete files

'"my_file.xlsx"

'"my_file2.xlsx"

'"my_name.xlsx"

'"my_n.xlsx"

'... an any file starting with "my_" and with an extension "xlsx"

VBA УДАЛЕНИЕ ФАЙЛОВ, СООТВЕТСТВУЮЩИХ РЕГУЛЯРНОМУ ВЫРАЖЕНИЮ

В некоторых случаях существуют более сложные правила, определяющие файлы, которые мы должны удалить. В таком случае мы можем создать новую функцию, которая использует регулярные выражения VBA :

Sub RegexKill(filePath As String, pattern As String)

    Dim regex As Object, fileName As String, deleted As Long

    Set regex = CreateObject("VBScript.RegExp")

    regex.pattern = pattern

    fileName = Dir(filePath)

    Do While fileName &lt;&gt; vbNullString

        If regex.Test(fileName) Then Kill (filePath &amp; fileName)

        fileName = Dir

    Loop

End Sub

Имея эти функции, мы можем удалять файлы, используя шаблоны (обязательно прочитайте мой учебник по регулярным выражениям VBA, если вы новичок в Regex).

Давайте использовать этот пример ниже. Я хотел бы удалить файлы «1.txt» и «2.txt» , сохранив оставшиеся файлы. Следовательно, я могу использовать шаблон Regex [0-9] , что означает любую цифру. Следовательно, выполнение будет выглядеть так:

RegexKill "C:\Users\tomasz.kacprowicz\Desktop\Test\", "[0-9].txt"

VBA УДАЛИТЬ ФАЙЛЫ РЕКУРСИВНО

В некоторых случаях вы можете не знать, в каком каталоге находится файл, который вы хотите удалить, и у вас есть только корневой каталог. Обход этих папок вручную является громоздким. Однако с помощью моей процедуры TraversePath , которую я создал в посте VBA Dir Function — How to traverse directorys, мы можем создать нашу собственную функцию RecursiveKill :

Sub RecursiveKill(filePath As String, fileName As String)

    Dim currDir As String

    Dim currentPath As String, dirItem As Variant

    Dim dirCollection As Collection

    Set dirCollection = New Collection

   

    'Delete the file in current Directory

    If Len(Dir(filePath &amp; fileName)) &gt; 0 Then

       SetAttr filePath &amp; fileName, vbNormal

       Kill filePath &amp; fileName

    End If

    'Delete the file recursively in remaining directories

    currDir = Dir(filePath, vbDirectory)

    Do Until currDir = vbNullString

        If Left(currDir, 1) &lt;&gt; "." And (GetAttr(filePath &amp; currDir) And vbDirectory) = vbDirectory Then

            dirCollection.Add filePath &amp; currDir &amp; "\"

        End If

        currDir = Dir()

    Loop

    

    For Each dirItem In dirCollection

        RecursiveKill CStr(dirItem), fileName

    Next dirItem

End Sub

Теперь, используя тот же пример, что и выше, давайте предположим, что у нас есть файл 1.txt , скопированный несколько раз в нашем каталоге. Мы можем легко удалить все копии следующим образом:

RecursiveKill "C:\some_path\", "1.txt"
