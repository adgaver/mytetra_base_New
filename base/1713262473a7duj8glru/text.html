Ведущий ноль — это любой 0, который вы добавляете перед первой ненулевой цифрой строки. По определению они невидимы, если значение имеет номер типа, поэтому требуется текстовое значение. Добавление ведущих нулей к числам в Power Query — полезный навык в нескольких случаях.

Если ваше значение требует фиксированной длины, ведущие нули могут предоставить вам надежный способ добиться этого. Ведущие нули также помогают при сортировке текстовых значений, содержащих числа. Это может быть полезно, поскольку текст, содержащий числа, часто сортируется странным образом. В этом посте я покажу вам три способа добавления ведущих нулей к числам.

Оглавление

1. Зачем добавлять ведущие нули?

Давайте посмотрим, когда полезны ведущие нули. Посмотрите, что происходит при сортировке следующего списка. Желаемый порядок сортировки числовых значений не представляет проблем:

= List.Sort( { 1, 11, 111, 20, 35 } ) // Result { 1, 11, 20, 35, 111 }


Как видите, результат сортируется от низкого к высокому ожидаемым образом. Все становится сложнее, когда вы сортируете текстовые значения, содержащие числа. Изучите следующие примеры:

Sorting text incl numbers:

= List.Sort( { "a1", "a11", "a20", "a111", "a35" } )

Result = { "a1", "a11", "a111", "a20", "a35" }

Desired = { "a1", "a11", "a20", "a35", "a111" }


Sorting text YYYY-M:

= List.Sort( { "2021-1", "2021-9", "2021-10", "2021-11" } )

Result = { "2021-1", "2021-10", "2021-11", "2021-9" }

Desired = { "2021-1", "2021-9", "2021-10", "2021-11" }


В двух предыдущих примерах функция List.Sort выполняет именно то, что вы указали, то есть сортирует в алфавитном порядке. В алфавитном порядке 111 стоит перед 20, а 11 перед 2. Ведь при алфавитной сортировке цифры идут после текста. И каждое число сортируется индивидуально. Вот почему цифры больше не отображаются в хронологическом порядке.

Тем не менее, вы можете сначала отсортировать значение по текстовой части и учитывать числовое значение, которое идет после. Итак, какое хорошее решение позволяет сортировать текстовые значения, но получать числа в нужном порядке?

К числам можно добавлять ведущие нули. Благодаря префиксу нулей ваш столбец ГГГГ-ММ снова хорошо сортируется.

Sorting text YYYY-MM:

= List.Sort( { "2021-01", "2021-09", "2021-10", "2021-11" } )

Result = { "2021-01", "2021-09", "2021-10", "2021-11" }


Если эта функция для вас новая, обязательно прочитайте подробно о функции List.Sort и различных способах ее сортировки.

2. Добавьте ведущие нули в числа

2.1. Использование объединения

Первый способ добавить ведущие нули — объединить текстовые значения. В настоящее время оба столбца имеют формат целых чисел. Для объединения данных они должны быть в текстовом формате. Шаг 1 – отформатировать номера месяцев по две цифры каждый. В пользовательском столбце вы можете написать:

= "0" & Text.From ( [Month] )


Это добавляет 0 перед каждым значением и превращает его в текст:

Как видно на рисунке, к числам 10–12 в начале также добавлен 0. Следующий шаг — извлечь два последних символа каждого текстового значения с помощью функции Text.End . Эта функция принимает в качестве аргументов:

  1. Текстовое значение
  2. Количество символов для извлечения

Ты можешь написать:

= Text.End( "0" & Text.From ( [Month] ), 2 )


Осталось добавить год в эту формулу. Тогда окончательная формула будет такой:

= Text.From( [Year] ) & "-" & Text.End( "0" & Text.From ( [Month] ), 2 )

2.2. Использование Text.PadStart

Другой способ добиться того же результата — использовать функцию Text.Padstart . Эта функция добавляет символы в начало текстового значения. Аргументы в пользу его написания:

  1. Текстовое значение, к которому нужно добавить символы.
  2. Количество символов, которое должно содержать преобразованное текстовое значение.
  3. Необязательный символ, добавляемый в начале текста.

Аргумент 3 является необязательным. Если он не указан, в качестве символа заполнения по умолчанию используется пробел .

В нашем примере мы хотим, чтобы в качестве символа заполнения использовался «0». Все значения столбца «Месяц» должны иметь длину 2 символа. Мы пробуем это:

= Text.PadStart( [Month], 2, "0" )


Power Query выдаст ошибку: «Expression.Error: мы не можем преобразовать значение 1 в тип Text».

Для функции требуется текстовое значение, тогда как столбец «Месяц» имеет числовой тип. Итак, по сути, мы добавляем в текст ведущие нули. Чтобы это исправить, вы можете изменить формулу на:

= Text.PadStart( Text.From( [Month] ), 2, "0" )


Осталось только добавить год в формулу, чтобы получить желаемый конечный результат:

= Text.From( [Year] ) & "-" & Text.PadStart( Text.From( [Month] ), 2, "0" )


2.3. Использование Number.ToText

Третий и простой способ — использовать функцию Number.ToText . Эта функция преобразует числовое значение в текстовое значение в указанном вами формате. Например, вы можете отформатировать число в процентах, присвоить ему экспоненциальное представление («4.0000000e+000»), а также добавить к числам ведущий 0.

Синтаксис функции следующий:

  1. числовое значение
  2. опционально форматированный ввод в виде текста
  3. необязательно, культура значения в виде текста

В нашем случае это означает, что следующий код может форматировать номера месяцев с ведущим нулем:

= Number.ToText( [Month], "00" ) )


Как и в предыдущих примерах, мы затем добавляем номер года, чтобы получить окончательную формулу:

= Text.From( [Year] ) & "-" & Number.ToText( [Month], "00" )


Заключение

Показанные методы могут быть полезны в определенных сценариях. Однако если у вас есть столбец даты в той же строке, вы можете предпочесть использовать другой метод, используя Date.ToText . Этот и другие приемы вы можете найти в этой статье о создании таблицы дат в Power Query.
