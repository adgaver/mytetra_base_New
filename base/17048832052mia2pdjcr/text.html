Вычислить N-й рабочий день от даты в Power Query

Рик — основатель BI Gorilla. Он считает, что обучение — одно из величайших удовольствий в жизни, и делится своими знаниями, чтобы помочь вам улучшить свои навыки.

В этой статье мы шаг за шагом покажем вам, как вычислить n-й рабочий день от даты в Power Query. Мы будем использовать смещение, которое также может учитывать праздничные и выходные дни.

Оглавление

Введение

В календарях или запросах, связанных с датами, обычно требуется получить рабочий день, который находится на расстоянии n дней от исходной даты.

Так как же его получить? В предыдущем посте я писал, как можно найти следующий рабочий день с помощью List.Generate . Этот пост идет еще дальше и показывает, как найти n-й рабочий день в месяце. Это включает в себя возможность заглянуть в будущее или назад во времени.

Некоторое время назад Имке написал способ получения N-го рабочего дня с помощью функций списка. В моей статье используется другой подход с использованием List.Generate , который дает тот же результат.

Решение в этом посте использует итерацию с List.Generate . При поиске N-го рабочего дня у нас есть дата начала. И начиная с этой начальной даты мы будем увеличивать или уменьшать количество рабочих дней с шагом в 1 день, пока не достигнем желаемого количества рабочих дней.

Как это могло сработать?

Получить N-й день недели

Допустим, вы хотите найти второй рабочий день после четверга, 15 сентября 2022 г. Вы хотите посчитать количество рабочих дней после вашей даты.

Итак, первый аргумент с начальным значением начинается с записи, содержащей поля:

  • Date , то есть дата, следующая за контрольной датой: Date.AddDays( #date ( 2022, 9, 15 ), 1 ) .
  • WD_Counter начинается с 0 и регистрирует, сколько рабочих дней содержит наш список.
  • IsWorkDay указывает, является ли текущее значение даты рабочим днем.

Следующий шаг — определить, когда List.Generate достиг n-го рабочего дня и может прекратить генерировать значения. Это происходит, когда мы обнаруживаем, что счетчик рабочих дней равен 2 и когда IsWorkDay равен true.

Если мы не проверим, является ли дата рабочим днем, а второй рабочий день приходится на пятницу, в противном случае запрос также вернет выходные дни.

Третий аргумент указывает List.Generate, как сгенерировать следующее значение для полей записи. В этом случае мы хотим:

  • Дата увеличения на 1 день
  • WD_Counter будет увеличиваться только в том случае, если дата равна рабочему дню.
  • IsWorkDay указывает true, если дата равна рабочему дню, и в противном случае — false.

List.Generate(

() => [ Date = Date.AddDays( #date( 2022, 9, 15 ), 1 ),

WD_Counter = 0,

IsWorkday = null

],

each if [WD_Counter] = 2 and [IsWorkday] = true then false else true,

each [

Date = Date.AddDays( [Date], 1 ),

WD_Counter = if Date.DayOfWeek( [Date], 1 ) < 5

then [WD_Counter] + 1 else [WD_Counter],

IsWorkday = Date.DayOfWeek( [Date], 1 ) < 5

]

)


На данный момент этот запрос возвращает список из 4 записей. Второй рабочий день после четверга, 15 сентября 2022 г., — понедельник, 19 сентября 2022 г. Запрос возвращает записи за 16–19 сентября 2022 г.

Поскольку нас интересует только дата второго дня недели после нашей контрольной даты, вы можете указать необязательный четвертый аргумент селектора List.Generate . Это возвращает список только дат.

А когда вы переносите список дат в функцию List.Last , она возвращает последнюю дату в списке, которая является нашим вторым днем ​​недели.

List.Last(

List.Generate(

() => [ Date = Date.AddDays( #date( 2022, 9, 15 ), 1 ),

WD_Counter = 0,

IsWorkday = null

],

each if [WD_Counter] = 2 and [IsWorkday] = true then false else true,

each [

Date = Date.AddDays( [Date], 1 ),

WD_Counter = if Date.DayOfWeek( [Date], 1 ) < 5

then [WD_Counter] + 1 else [WD_Counter],

IsWorkday = Date.DayOfWeek( [Date], 1 ) < 5

],

each [Date]

)

)


Эта формула возвращает 19 сентября 2022 года, день, который мы искали. В пользовательском интерфейсе эти три шага выглядели следующим образом:

Получить N-й рабочий день

Если вы хотите вернуться на N-й рабочий день, ваша функция также должна учитывать праздничные дни. Давайте представим, что мы продолжаем наш предыдущий пример и хотим указать, что 19 сентября 2022 года является выходным днем.

Единственное, что нам нужно изменить, это:

  • WD_Counter не должен увеличиваться в праздничные дни
  • IsWorkday должен указывать false в праздничные дни.

List.Last(

List.Generate(

() => [ Date = Date.AddDays( #date( 2022, 9, 15 ), 1 ),

WD_Counter = 0,

IsWorkday = null

],

each if [WD_Counter] = 2 and [IsWorkday] = true then false else true,

each [

Date = Date.AddDays( [Date], 1 ),

WD_Counter = if Date.DayOfWeek( [Date], 1 ) < 5

and not List.Contains( {#date( 2022, 9, 19 )}, [Date] )

then [WD_Counter] + 1 else [WD_Counter],

IsWorkday = Date.DayOfWeek( [Date], 1 ) < 5

and not List.Contains( {#date( 2022, 9, 19 )}, [Date] )

],

each [Date]

)

)


Благодаря этой корректировке код возвращает вторник, 20 сентября 2022 г., как второй рабочий день после 15 сентября 2022 г.

Поскольку теперь мы используем список праздников несколько раз и можем захотеть ссылаться на более длинный список праздников, хорошей практикой будет превратить этот код в переменные.

let

Holidays = {#date( 2022, 9, 19 )},

ListOfDates =

List.Generate(

() => [ Date = Date.AddDays( #date( 2022, 9, 15 ), 1 ),

WD_Counter = 0,

IsWorkday = null

],

each if [WD_Counter] = 2 and [IsWorkday] = true then false else true,

each [

Date = Date.AddDays( [Date], 1 ),

WD_Counter = if Date.DayOfWeek( [Date], 1 ) < 5

and not List.Contains( Holidays , [Date] )

then [WD_Counter] + 1 else [WD_Counter],

IsWorkday = Date.DayOfWeek( [Date], 1 ) < 5

and not List.Contains( Holidays, [Date] )

],

each [Date]

),

Result = List.Last( ListOfDates )

in

Result


Прошлые и будущие рабочие дни

Если вы хотите заглянуть в прошлое на N-й рабочий день от исходной даты, код потребует нескольких корректировок.

Чтобы найти 5-й рабочий день до контрольной даты, вам необходимо скорректировать следующее:

  • Поле даты в аргументе 1 определяет первую дату проверки. Следует вернуться к второму аргументу Date.AddDays , который должен быть -1 вместо +1.
  • Условие WD_Counter в аргументе 2 и измените его на 5.
  • Поле даты в аргументе 3 определяет следующее значение даты. Он должен оглянуться назад, как в аргументе 1.

Вместо изменения этих переменных вручную мы могли бы использовать код, который учитывает смещение рабочего дня. Давайте введем несколько новых переменных:

  • WorkDayOffset определяет, на сколько дней вы хотите просмотреть назад или вперед.
  • AddDays возвращает +1 для положительных значений WorkDayOffsets и -1 для отрицательных значений смещения. Мы используем это в функции Date.AddDays в строках 8 и 15, чтобы определить, должно ли следующее значение увеличиваться или уменьшаться.
  • NumOfWDs означает количество рабочих дней, которые мы хотим иметь в нашем списке. Number.Abs гарантирует, что это число всегда будет положительным.

Настройка этого в нашем коде выглядит следующим образом:

let

WorkDayOffset = -5,

AddDays = Number.Sign( WorkDayOffset ),

NumOfWD = Number.Abs( WorkDayOffset ),

Holidays = {#date( 2022, 9, 19 )},

ListOfDates =

List.Generate(

() => [ Date = Date.AddDays( #date( 2022, 9, 15 ), AddDays ),

WD_Counter = 0,

IsWorkday = null

],

each if [WD_Counter] = NumOfWD

and [IsWorkday] = true then false else true,

each [

Date = Date.AddDays( [Date], AddDays ),

WD_Counter = if Date.DayOfWeek( [Date], 1 ) < 5

and not List.Contains( Holidays , [Date] )

then [WD_Counter] + 1 else [WD_Counter],

IsWorkday = Date.DayOfWeek( [Date], 1 ) < 5

and not List.Contains( Holidays, [Date] )

],

each [Date]

),

RelevantWorkDay= List.Last( ListOfDates )

in

RelevantWorkDay


Окончательные корректировки

Теперь все готово для большинства ваших требований. Тем не менее, есть некоторые небольшие изменения, чтобы сделать код более надежным и более похожим на функцию Excel РАБДЕНЬ. Нам все еще необходимо внести изменения для следующего рассмотрения:

  • Пользователи могут захотеть или не захотеть использовать Holidays. Чтобы дать им возможность использовать или пропускать праздники, нам нужно убедиться, что наша функция может обрабатывать оба сценария. В настоящее время наш код проверяет, попадает ли сгенерированное значение даты в список праздников. Если пользователь опускает этот аргумент, мы можем предоставить функции List.Contains пустой список {}. List.Contains ( {}, [Date] ) всегда будет возвращать false и, следовательно, не влияет на проверку того, выпадает ли дата на праздничный день.
  • При использовании аргумент Holidays будет вызываться несколько раз. Из соображений производительности мы будем буферизовать эти значения в памяти с помощью List.Buffer .
  • Третий аргумент дважды проверяет, является ли следующее значение рабочим днем. Один раз для WD_Counter и один раз для поля IsWorkDay. Мы будем использовать переменную для оценки этого только один раз.
  • Функция Excel РАБДЕНЬ возвращает введенную дату, когда пользователь вводит 0 в качестве WorkDayOffset. Мы добавим аргумент, который возвращает базовую дату, когда пользователь вводит значение смещения, равное 0.

let

WorkDayOffset = -5,

AddDays = Number.Sign( WorkDayOffset ),

NumOfWD = Number.Abs( WorkDayOffset ),

Holidays = {#date( 2022, 9, 19 )},

BufferedHolidays = List.Buffer( Holidays ?? {} ),

ListOfDates =

List.Generate(

() => [ Date = Date.AddDays( #date( 2022, 9, 15 ), AddDays ),

WD_Counter = 0,

IsWorkday = null

],

each if [WD_Counter] = NumOfWD

and [IsWorkday] = true then false else true,

each

let

NextDateIsWD = Date.DayOfWeek([Date], 1)

< 5 and not List.Contains( BufferedHolidays, [Date])

in

[

Date = Date.AddDays( [Date], AddDays ),

WD_Counter = if NextDateIsWD then [WD_Counter] + 1

else [WD_Counter],

IsWorkday = NextDateIsWD

],

each [Date]

),

RelevantWorkDay = List.Last( ListOfDates ),

Result = if WorkDayOffset = 0 then #date( 2022, 9, 15 )

else RelevantWorkDay

in

Result


Выполнив эти шаги, вы теперь увидели всю логику, необходимую для подготовки финальной функции для получения вашего N-го рабочего дня.

Код функции для N-го рабочего дня

С некоторыми небольшими изменениями в коде вы получите следующую функцию.

let func =

( ReferenceDate as date, WorkdayOffset as number, optional Holidays as list) =>

let

/* Debug parameters

ReferenceDate = #date( 2021,1,1 ),

WorkdayOffset = 1,

Holidays = { #date( 2021,1,1 ) },

*/

// Returns empty list if Holidays are omitted

BufferedHolidays = List.Buffer( Holidays ?? {} ),

// Determines the positive or negative offset of Date.AddDays

AddDays = Number.Sign( WorkdayOffset ),

// Creates the List of Dates until it reaches the relevant Workday

ListOfDates =

List.Generate(

() => [

Date = Date.AddDays( ReferenceDate, AddDays ),

WD_Counter = 0,

IsWorkday = null

],

each if [WD_Counter] = Number.Abs( WorkdayOffset )

and [IsWorkday] = true then false else true,

each

let

NextDateIsWD = Date.DayOfWeek([Date], 1)

< 5 and not List.Contains( BufferedHolidays, [Date])

in

[

Date = Date.AddDays([Date], AddDays ),

WD_Counter = if NextDateIsWD then [WD_Counter] + 1

else [WD_Counter],

IsWorkday = NextDateIsWD

],

each [Date]

),

// Returns the relevant Workday, which is always last in the list

RelevantWorkday = List.Last( ListOfDates ),

// When 0 is used as Workdayoffset, the function returns the referencedate

Result = if WorkdayOffset = 0 then ReferenceDate else RelevantWorkday

in

Result,

documentation = [

Documentation.Name = " WORKDAY ",

Documentation.Description = " Function returns the date that lies an specfied number of business days from the reference date. ",

Documentation.LongDescription = " Function returns the date that lies an specfied number of business days from the reference date. The function takes into account weekends and can optionally respect a list of holiday dates. ",

Documentation.Category = " Nth Business Day ",

Documentation.Source = " https://gorilla.bi – https://gorilla.bi/power-query/nth-business-day-from-date/ ",

Documentation.Version = " 1.0 ",

Documentation.Author = " Rick de Groot",

Documentation.Examples = {[Description = " ",

Code = " let

ReferenceDate = #date( 2022, 9, 15 ),

WorkdayOffset = 2,

Holidays = { #date( 2022, 9, 19 ) },

Result = WORKDAY( ReferenceDate, WorkDayOffset, Holidays )

in

Result ",

Result = " #date(2022, 9, 20)

"]}]

in

Value.ReplaceType(func, Value.ReplaceMetadata(Value.Type(func), documentation))


Функция имеет всю логику для поиска N-го рабочего дня. Она работает аналогично функции Excel РАБДЕНЬ, при этом параметры ведут себя следующим образом:

  • Базовая дата — это начальная дата, начиная с которой функция будет искать n-й рабочий день. n-й рабочий день может быть положительным или отрицательным числом.
  • WorkdayOffset определяет , на сколько рабочих дней вы хотите просмотреть вперед или назад. Ввод 5 означает возврат даты, которая находится на 5 рабочих дней в будущем, а -5 вернет дату, которая будет на 5 рабочих дней назад во времени.
  • Аргумент Holidays позволяет ввести список дат. Самый простой способ сохранить это — добавить отдельный запрос, содержащий ваши даты. Затем вы можете сослаться на этот столбец даты в этом запросе, написав TableName[ColumnName]. Этого достаточно, чтобы функция уважала те времена.

В своем календаре вы сможете увидеть в действии функцию вашего n-го рабочего дня:

Теперь вы можете легко найти такие показатели, как:

// Formula retrieves the 3rd business day from the date

= WORKDAY( [Date], 3)

// Formula retrieves the 3rd business day of the month the current row's date is in

= WORKDAY( Date.AddDays( Date.StartOfMonth( [Date], -1 ) ), 3, Holidays )

// Retrieves the day that is 3 working days after the first of the month

= WORKDAY( Date.StartOfMonth( [Date]), 3, Holidays )

// nth business day of Week

= WORKDAY( Date.StartOfWeek( [Date] ) , 3, Holidays )
