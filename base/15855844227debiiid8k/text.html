Создание и публикация базы данных Access в SharePoint

Access 2016 Access 2013 Access 2010

Access 2010 и службы Access на сайте SharePoint можно использовать для создания приложений веб-базы данных. Эта возможность позволяет:

  • организовать доступ к данным и обеспечить его безопасность;
  • предоставлять общий доступ к корпоративным данным через интрасеть или Интернет.

    • Примечание:  Для работы с веб-базой данных необходима учетная запись пользователя. Анонимный доступ не поддерживается.

  • создавать приложения для работы с базами данных, не требующие использования Access.

В этой статье приведены общие сведения о проектировании веб-баз данных Access. Раздел См. также предоставляет ссылки на статьи, которые содержат введение в проектирование баз данных для настольных компьютеров.

Примечания: 

  • Если эти сведения не соответствуют тому, что вы видите, вероятно, вы пользуетесь обновленной версией службы Office 365. Чтобы найти необходимую вам информацию, см. статью Редактирование и публикация веб-базы данных в SharePoint 2010 .

Создание веб-базы данных

Обзор

Access 2010 и службы Access предоставляют платформу для создания баз данных, которые можно использовать на сайте SharePoint. Вы можете проектировать и публиковать веб-базу данных с помощью Access 2010 и служб Access, а пользователи, обладающие учетными записями на сайте SharePoint, могут использовать ее в веб-браузере.

Принцип работы

При публикации веб-базы данных службы Access создают сайт SharePoint, на котором она размещается. Все объекты и данные базы попадают в списки SharePoint на этом сайте. В видеоролике ниже показан этот процесс.
