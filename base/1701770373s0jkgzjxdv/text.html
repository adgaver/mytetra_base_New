Pandas ExcelFile.Parse – все, что вам нужно знать

Арулиус Савио / 27 февраля 2023 г.

Когда дело доходит до анализа данных, основной задачей является импорт данных в инструмент, который их анализирует. Сложная часть заключается в импорте данных, поскольку не существует универсального формата для консолидации данных. Некоторые предпочитают электронные таблицы, а другие довольствуются файлами формата «.csv». Некоторые могут даже импортировать данные непосредственно с веб-сайтов, а другие предпочитают их в виде файлов «.txt».

Учитывая множество способов импорта данных, инструмент анализа данных должен быть совместим с различными форматами доступных хранилищ данных. В этой статье основное внимание уделяется одному такому хранилищу, которое повсеместно используется для записи данных – MS Excel. Мы импортируем данные файла MS Excel с помощью функции ExcelFile.Parse() из библиотеки pandas .

Итак, давайте начнем с импорта библиотеки pandas , используя приведенный ниже код.


import pandas as pd


После этого мы углубимся в работу функции ExcelFile.Parse() в каждом из следующих разделов.

Синтаксис функции ExcelFile.Parse( )

Ниже приведен синтаксис, содержащий обязательные и необязательные конструкции, необходимые для правильного функционирования функции ExcelFile.Parse() .


pandas.ExcelFile.Parse(sheet_name=0, header=0, names=None,

index_col=None, usecols=None, sqeeze=None, converters=None,

true_values=None, false_values=None, skiprows=None,

nrows=None, na_values=None, parse_dates=False,

date_parser=None, thousands=None, comment=None,

skipfooter=0, convert_float=None, mangle_dupe_col=True, **kwds)


где,

имя_листа – имя или номер страницы листа в файле MS Excel, содержащего импортируемые данные.

заголовок — по умолчанию установлен в ноль (0). Он используется для указания строки, содержащей метки данных, подлежащих анализу.

имена — по умолчанию установлено значение «Нет», оно используется для указания имен столбцов, которые будут использоваться.

index_col – по умолчанию установлено значение «Нет», оно используется для указания столбца, который будет использоваться в качестве меток строк для анализируемых данных.

usecols — используется для возврата подмножества входных столбцов в зависимости от того, установлена ​​ли строка или список строк/числ. Поскольку по умолчанию для него установлено значение «Нет», он также может быть установлен как вызываемый.

сжатие — по умолчанию установлено значение «Нет». Оно используется для возврата серии, когда анализируемые данные содержат только один столбец, но он устарел в версиях, следующих за 1.4.0.

конвертеры — по умолчанию установлено значение «Нет», оно используется для указания набора функций, которые могут скрывать значения в определенных столбцах.

true_values ​​— по умолчанию установлено значение «Нет», оно используется для указания значений, которые должны считаться «Истина» при анализе данных.

false_values ​​— по умолчанию установлено значение «Нет», оно используется для указания значений, которые должны считаться «Ложными» при анализе данных.

Skiprows – по умолчанию установлено значение «Нет», оно используется для указания номеров строк или количества строк, которые нужно пропустить с начала файла.

nrows — по умолчанию установлено значение «Нет», оно используется для указания количества строк, подлежащих анализу.

na_values ​​— по умолчанию установлено значение «Нет», оно используется для указания строк, которые следует рассматривать как значения NA, например «#N/A», «#NA», «NaN», «na» и т. д.

parse_dates – по умолчанию установлено значение «False». Оно используется для указания списка дат во входных данных, которые необходимо проанализировать.

date_parser – по умолчанию установлено значение «Нет», оно используется для преобразования столбцов, содержащих строки, в массив формата «datetime».

тысячи — по умолчанию установлено значение «Нет», оно используется для включения разделителя тысяч для значений столбца, хранящихся в формате «Текст» во входных данных.

комментарий – по умолчанию установлено значение «Нет», оно используется для выделения местоположений комментариев во входном файле MS Excel путем передачи символов.

skipfooter — по умолчанию установлено значение 0 (0), оно используется для пропуска строк в конце входных данных.

Convert_float – по умолчанию установлено значение «Нет», оно используется для преобразования данных, хранящихся как «float», в «int», но не рекомендуется в версиях после 1.3.0.

mangle_dupe_col – по умолчанию установлено значение «True», оно используется для указания повторяющихся столбцов как «X», «X.1»,…. «X.N» & используется для перезаписи данных в повторяющихся столбцах, если установлено значение «False». Он устарел в версиях после 1.5.0.

**kwds – kwargs или аргумент ключевого слова, который представляет собой необязательную конструкцию, используемую для передачи переменной длины аргумента ключевого слова в функцию.


Функция ExcelFile.Parse() объяснена на примере

Давайте сначала укажем местоположение файла, содержащего данные, которые необходимо проанализировать с помощью функции ExcelFile() , как указано ниже.


XL = pd.ExcelFile('D:\Ask Python\Examples.xlsx')


После этого пришло время проанализировать данные в этом файле, в частности, интересующие данные — это те, которые попадают в первые три столбца первого листа. Это можно сделать, используя следующий код.


df = XL.parse('Sheet1', usecols = [1, 2, 3])

print (df)


Печать кадра данных дает следующий результат.

Выбранный кадр данных перенастроен

Теперь предположим, что мы хотим, чтобы самый первый столбец вывода, столбцы с «0, 1, 2 и 3», был заменен содержимым столбца второго столбца, тогда то же самое можно сделать следующим образом.


df = XL.parse('Sheet1', usecols = [1, 2, 3], index_col = 0)

print (df)


Модифицированный фрейм данных

Предположим, что нужны только результаты первых трех строк, для этого можно использовать опцию nrows .


df = XL.parse('Sheet1', usecols = [1, 2, 3], index_col = 0, nrows = 3)

print (df)


Кадр данных с анализом только 3 строк
